Azure Portal、SMB、または REST を使用して Azure ファイル共有に加えられた変更は、サーバー エンドポイントに対する変更のように検出やレプリケートは即座には行われません。 Azure Files にはまだ変更通知/ジャーナルがなく、ファイルが変更されたときに自動的に同期セッションを開始する方法がないため、変更はすぐにはレプリケートされません。 Windows Server では、Azure File Sync は [Windows USN ジャーナル](https://msdn.microsoft.com/library/windows/desktop/aa363798.aspx)を使用して、ファイルが変更されたときに同期セッションを自動的に開始します。

Azure ファイル共有に対する変更を検出するために、Azure File Sync には「変更検出ジョブ」と呼ばれるスケジュールベースのジョブが用意されており、ファイル共有内のすべてのファイルを列挙し、そのファイルの同期されたバージョンと比較します。 変更検出ジョブがファイルが変更されたと判断すると、Azure File Sync が同期セッションを開始します。 変更検出ジョブは 24 時間ごとに実行されます。 変更検出ジョブは Azure ファイル共有内のすべてのファイルを列挙することで機能するため、名前空間の規模が大きいほど時間がかかります。 つまり、これらの名前空間では、変更されたファイルを判断するのに 24 時間ごとに 1 回より長くなる場合があります。 

長期的には、Windows Server における USN と同様の変更検出機能を Azure File にも追加することを検討しています。 今後この機能の開発を優先的に進めるために、[Azure Files UserVoice](https://feedback.azure.com/forums/217298-storage/category/180670-files) で投票をお願いします。
