# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>新しいクラウド パートナー ポータルに移行する方法

クラウド パートナー ポータルへの移行方法は大きく進歩しました。 発行プロセスに関するフィードバックを受けて多額の投資を行い、使いやすさの向上と発行エクスペリエンス全体の改善を図りました。 
*    組織 ID ログインのサポート
*    リアルタイムのフィールド検証によるエラー検出
*    発行ワークフローの簡素化

プランの移行を非常に簡単に行うことができるようになりましたので、使いやすさが増した発行を試す準備ができている方は、必要な作業をご確認ください。

## <a name="what-do-you-need-to-do"></a>必要な作業
移行を成功に導くには、次の手順にしたがってください。

**移行のステージ**
|          |          |
|----------|----------|
|1.  準備が整っていることを確認する | プランをクラウド パートナー ポータルへ移行する前に、作業の内容を十分理解しておく必要があります。<br/><br/>**すべてのプランが移行の対象になりますか?**<br/>現時点では、すべての仮想マシン プランが移行の対象になります。 つまり、ドラフト、ステージング中、および掲載中の仮想マシン プランが移行されます。 <br/>他の種類のプラン (ソリューション テンプレート、体験版など) はすべて、当面のところ、これまで通り古いポータルで動作します。<br/><br/>**ライブ プランは引き続き動作しますか?**<br/>はい。この移行によるライブ プランへの影響はありません。 また、お客様の発行者アカウント情報が事前に入力されているため、移行が完了したらすぐに新しいポータルでプランを更新できます。 ステージング中のプランはドラフトになるため、新しいポータルで発行する必要があります。<br/><br/>**移行するときにプランの価格を変更できますか?**<br/>いいえ。すべてのライブ プランについて、この移行では価格を変更できないため、価格は完全に同じままにする必要があります。<br/><br/>**プランがステージング中または発行中の場合どうなりますか?**<br/>古いポータルに発行中またはステージング中のプランがあると、移行は失敗します。 移行するときは、オファーが発行済みまたはステージング済みの状態であるようにする必要があります。<br/><br/>**新しいロールベースのアクセス制御はどのように機能しますか?**<br/>新しいポータルでは、管理者が発行プロファイルの所有者になり、すべての共同管理者が共同作成者になります。 発行プロファイルを編集する場合は、新しいポータルを使用する必要があります。 <br/>ユーザーを管理するために、古いポータルのプランに関する作業を行うユーザーを追加する場合は、古いポータルの共同管理者として追加します。 新しいポータルのプランに関する作業を行うユーザーを追加する場合は、新しいポータルの共同管理者として追加します。<br/><br/>**移行を行うと最終版になりますか?**<br/>プランが移行されると、プランに関する古いポータルでのアクションはすべて読み取り専用になり、古いポータルでの作成は行うことができなくなります。 移行したすべてのプランの編集や操作は中断した時点から再開できますが、作業を行うことができる場所は新しいポータルに限られます。|
|2.  移行を申請する | 移行の準備が整ったら、クラウド パートナー ポータルの [学習] ページで、お客様の発行者名と ID を使用して移行を申請します。![発行のフロー図](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)|
|3.  日付を確認する | プランは、毎週火曜日の午前 10 時から午後 1 時 (太平洋標準時) の間に移行されます。転送開始の 24 時間前までに、発行アカウントの所有者に転送時刻の確認が求められます。 現在のワークフローに影響を与えないよう、移行を別の時間に行う必要がある場合は、最適な時間に移行できるよう調整いたします。<br/>移行が完了し、新しいポータルで作業を再開できるようになったことを知らせるメール通知が送られます。|
|4.  新しい環境を利用する | 移行が終了したら、新しいポータルを使ってみましょう。 これ以降、プランは古いポータルでは読み取り専用になり、作成や編集はすべて新しいポータルから実行します。 <br/><br/>**移行時に問題が発生しました。どうすればよいですか?** 移行が失敗した場合は、変更を元に戻し、別の日時に移行スケジュールを再設定します。 移行の終了後に質問がある場合は、フィードバック ツールからお問い合わせください。|
