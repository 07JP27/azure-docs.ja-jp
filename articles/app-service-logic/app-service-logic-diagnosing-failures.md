# Logic Apps の診断

Logic App に問題が発生した場合、いくつかの方法でその原因を理解することができます。

## 管理ポータル ツール

Azure ポータルには、各段階で各 Logic App を診断するためのツールがあります。

### トリガー履歴

各 Logic App には少なくとも 1 つのトリガーがあります。アプリが起動しないことに気付いた場合、最初にトリガー履歴を調べます。トリガー履歴にはここにある Logic App メイン ブレードからアクセスできます。

![][1]

これには Logic App によるすべてのトリガー試行が一覧表示されます。いずれかのエントリをクリックすると、次のレベルの詳細が表示されます (具体的には、トリガー試行で生成された入力または出力が表示されます)。'Failed (失敗)' トリガーが見つかったら、そのトリガー試行をクリックし、出力リンクを表示してください。エラー メッセージが生成された原因を理解できることがあります (例: 無効な FTP 資格情報)。

表示されるステータス:

* スキップ済み - エンドポイントにポーリングを実行してデータが利用できるか確認し、利用できるデータがないという応答を得ました。
* 成功 - データは利用できるという応答をトリガーは受けました。この出所には、手動トリガー、繰り返しトリガー、ポーリング トリガーの可能性があります。多くの場合、これには 'Fired (起動)' が付随しますが、条件やコード ビューの splitOn が十分ではない場合、付随しません。
* 失敗 - エラーが発生しました。

#### トリガーを手動で開始する

Logic App で利用できるトリガーをすぐに確認するには (次の発生を待たずに)、メイン ブレードの **[トリガーの選択]** ボタンをクリックすれば、いつでも確認を強制できます。たとえば、Dropbox トリガーでこれをクリックすると、新しいファイルに関するポーリングを Dropbox に実行するワークフローがすぐに開始されます。

### 実行履歴

トリガーが起動されるたびに、実行が記録されます。実行にはメイン ブレードからアクセスできます。これには、フローで発生したことを理解するために役立つさまざまな情報が含まれています。

![][2]

実行には次のステータスが与えられます。

* 成功 - すべてのアクションが成功しました。あるいは、エラーが発生しましたが、ワークフローの後続のアクションにより処理されました (つまり、'Failed (失敗)' アクション後に実行するアクションが設定されていました)。
* 失敗 - 少なくとも 1 つのアクションに失敗し、それがワークフローの後続のアクションにより処理されませんでした。
* キャンセル - ワークフローは実行されていましたが、キャンセル要求を受け取りました。
* 実行中 - ワークフローは現在実行中です。現行の App Service プランで調整中のフローにも発生することがあります。詳細については、[価格設定ページ](https://azure.microsoft.com/pricing/details/app-service/plans/)のアクション制限を参照してください診断の構成 (実行履歴の下のグラフ) からは、発生したスロットル イベントについても理解できます。

実行ページでは、さらに詳しい情報にアクセスできます。

#### トリガー出力

トリガー 出力には、トリガーから受け取ったデータが表示されます。すべてのプロパティが予想どおり返されているかどうかを理解するために役立つことがあります。

>[AZURE.NOTE] 不明なコンテンツが表示される場合に、Logic Apps による[さまざまなコンテンツ タイプの処理](app-service-logic-content-type.md)方法を理解するために役立つことがあります。

![][3]

#### アクションの入力と出力

アクションが受け取った入力と出力の詳細にアクセスできます。これは出力の大きさと形を理解するために役立ちます。エラー メッセージが生成されていれば、それも確認できます。

![][4]

## ワークフロー ランタイムのデバッグ

入力、出力、実行のトリガーを監視するだけでなく、デバッグに役立つ手順をワークフローに追加すると場合によっては効果的です。ワークフローに手順として追加できる強力なツールの 1 つが [RequestBin](http://requestb.in) です。RequestBin では、HTTP 要求の大きさ、形、書式を正確に理解する目的で、HTTP 要求インスペクターを設定できます。新しい RequestBin を作成し、URL を Logic App HTTP POST アクションに貼り付けることができます。本文にはあらゆるコンテンツ (式や別の手順の出力など) を指定し、テストできます。Logic App の実行後、RequestBin を更新し、Logic App エンジンから生成されたとき、要求がどのように書式設定されたかを確認できます。
