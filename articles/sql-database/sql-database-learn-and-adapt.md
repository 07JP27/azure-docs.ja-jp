# <a name="azure-sql-database-learns-amp-adapts"></a>Azure SQL Database の学習 &amp; 適応
## <a name="how-azure-sql-database-learns-adapts-and-grows-with-your-application"></a>Azure SQL Database でアプリケーションを学習し、適応し、共に成長する方法
Azure SQL Database は、1 日に数十億単位のトランザクションと百万単位のデータベースを管理しています。 また、SQL Database の主な機能の 1 つとして、常にアプリを学習し、適応することがあります。 そのため、手動の作業は最小限で、パフォーマンス、信頼性、データ セキュリティを動的に最大化することができます。

![learn-and-adapt](./media/sql-database-learn-and-adapt/sql-database-learn-and-adapt.png)

## <a name="threat-detection-and-alerts"></a>脅威の検出とアラート
SQL Database の脅威と異常の検出機能には、行動分析、リアルタイムのアラート、構成可能な脅威ポリシー、監査ログなど、通常とは異なるパターンを検出して修正する高度な組み込みの機能があります。

[ドキュメントを参照する](sql-database-threat-detection-get-started.md)

## <a name="automatic-tuning"></a>自動チューニング
SQL Database には、最適なパフォーマンスでアプリが実行される機能が備わっています。 アプリのパターンを継続的に学習し、それに合わせてパフォーマンスを自動チューニングし、自動的に改善します。Microsoft は望ましくない状況を把握しているので、ユーザーは何もする必要はありません。

[ドキュメントを参照する](http://go.microsoft.com/fwlink/?LinkID=787566)

## <a name="insights-when-you-need-them"></a>必要に応じて洞察
SQL Database では、各クエリとその期間、頻度、リソース使用率を追跡できます。 このテレメトリに基づき、自動アルゴリズムによって、クエリに合わせてデータベースが最適に調整されます。 また、SQL Database から洞察が提供されるので、クエリをチューニングし、パフォーマンスの問題を解決する時間を最短にすることができます。 つまり、リソース消費、上位のパフォーマンスのクエリ、ドリルダウンして詳細を取得する機能について、直接洞察することができます。

[ドキュメントを参照する](http://go.microsoft.com/fwlink/?LinkID=787567)

## <a name="no-administration-required"></a>管理が不要
SQL Database には、自動管理機能と、アプリに必要なデータ保護機能があります。そのため、最も望ましいこと、つまりコーディングに集中できるようになります。 バックアップ、障害復旧時のフェールオーバー、インフラストラクチャのメンテナンス、セキュリティとソフトウェアの修正プログラム、機能の更新も自動的に行うことができます。 SQL Database がバックグラウンドで動作しているときにお金と時間を節約できるだけでなく、恐ろしいアプリのダウンタイムを心配する必要がなくなります。

[ドキュメントを参照する](http://go.microsoft.com/fwlink/?LinkID=787568)
