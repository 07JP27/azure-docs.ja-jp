# エラスティック データベース クエリ - 2015 年 6 月プレビュー 

エラスティック データベース クエリは、2015 年 6 月末の Azure SQL Database プレビューの新機能です。Azure SQL Database への単一接続を使って、複数のデータベースにまたがる T-SQL クエリを実行する機能が導入されました。これにより、エラスティック データベースのシャード マップを使って定義される一連のシャード化されたデータを、単一の統合データ ストアとして表示できます。レポートとデータ統合の目的で使用される、すべての分散クエリ処理はバック グラウンドで処理されます。ADO.Net、ODBC、JDBC、Node.js、PHP などを含む、SQL Database と通信できる任意のドライバーを使用した接続がサポートされています。

## エラスティック データベース クエリのシナリオ

エラスティック データベース クエリの目的は、複数のデータベースやシャードにより、T-SQL クエリの 1 つの全体的な結果に行を使用できるシャード化されたデータ間のレポート シナリオを容易にすることです。T-SQL クエリは、ユーザーまたはアプリケーションによって直接、またはグローバル クエリのエラスティック クエリ データベースに接続されているツールを使って間接的に構成できます。これは、商用の BI、レポート、データ統合ツール、エラスティック スケール ライブラリを使って直接拡張できないパッケージ ソフトウェアに特に便利です。また、SQL Server Management Studio や Visual Studio によって発行されるクエリ経由でシャード化されたデータのコレクション全体に簡単にアクセスでき、Entity Framework やその他の ORM 環境からの複数のシャードの透過的なデータ アクセスをサポートします。図 1 では、グローバル クエリの Elastic Database クエリによって、既存のクラウド アプリケーション (エラスティック拡張ライブラリが使用される場合があります) だけでなく、シャード化されたデータ層にクエリを発行する 2 番目の異なる方法を提供するこのシナリオを示しています。

![エラスティック データベース クエリ][1]

シナリオは、次のトポロジで特徴付けることができます。

-	シャード化されたデータ層 (トポロジ 1): データ層はシャーディング用に設計されています。シャーディングは、(1) エラスティック データベースのクライアント ライブラリまたは (2) 自己シャーディングを使って実行、管理されます。このレポート ケースは、エラスティック データベースのクライアント ライブラリで複数シャードのクエリが実現できる以上の機能性や接続要件で、多くのシャード間のレポートをコンピューティングすることです。 

-	複数データベースの設計 (トポロジ 2): ここでは、さまざまな種類のデータが異なるデータベースに存在するよう、データ層が垂直方向にパーティション分割されています。このレポート ケースは、複数のデータベースに分散され、ただしレポートの全体的な結果は 1 つに統合される必要のあるデータのレポートを計算することです。

エラスティック データベース クエリのプレビューは、どちらの場合も対象としています。トポロジ 1 を使用する顧客は、エラスティック データベースのクライアント ライブラリを使ってシャードを管理する場合、既存のシャード マップを利用できます。セルフ シェーダーは、このプレビュー機能を使うために、データ層のエラスティック データベース ツールを使って、シャード マップを作成する必要があります。トポロジ 2 を使用する顧客は、各データベースのさまざまなシャード マップを作成する必要があります。これらのシャードのマップは、クロスシャードのクエリにテーブルを公開できる 1 つのシャードのみを指し示します。
