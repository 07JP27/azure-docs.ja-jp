# <a name="policy-requirements-for-pci-dss-compliant-environments"></a>PCI DSS に準拠する環境のポリシー要件  
## <a name="pci-dss-requirement-12"></a>PCI DSS 要件 12

**すべての担当者の情報セキュリティに対応するポリシーを管理**

> [!NOTE]
> これらの要件は、[PCI データ セキュリティ基準 (DSS) バージョン 3.2](https://www.pcisecuritystandards.org/document_library?category=pcidss&document=pci_dss) の一部として [Payment Card Industry (PCI) Security Standards Council](https://www.pcisecuritystandards.org/pci_security/) によって定義されています。 各要件のテスト手順およびガイダンスについては、PCI DSS をご覧ください。

強力なセキュリティポリシーは、事業体全体でのセキュリティの方向性を設定し、担当者に対して期待される内容を示します。 すべての担当者は、データの機密性とその保護に関する自身の責任を認識する必要があります。 要件 12 において、"担当者" とは、フルタイムおよびパートタイムの従業員、一時的な従業員、事業体の敷地内に "常駐" しているか、またはカード会員データ環境にアクセスできる請負業者やコンサルタントのことです。

## <a name="pci-dss-requirement-122"></a>PCI DSS 要件 12.2

**12.2** 以下のようにリスク アセスメント プロセスを実装します。
- 少なくとも年 1 回および環境に大きな変更があった場合 (買収、合併、移転など) に実施します。
- 重要な資産、脅威、脆弱性を識別します。
- 結果として文書化された正式なリスク分析を生成します。
- > リスク アセスメント方法の例としては、OCTAVE、ISO 27005、および NIST SP 800-30 がありますが、これらに限定されません。

**責任:&nbsp;&nbsp;`Customer Only`**

|||
|---|---|
| **プロバイダー <br />(Microsoft&nbsp;Azure)** | 適用不可。 |
| **お客様<br /> (PCI&#8209;DSS&nbsp;Blueprint)** | お客様は、要件 12.2 に示されているすべての脅威に対処するリスク アセスメント プロセスの実装に責任を持ちます。|
