# <a name="azure-government-compliance"></a>Azure Government のコンプライアンス
## <a name="blueprint-customer-responsibilities-matrix-streamlining-ato-processes"></a>Blueprint Customer Responsibilities Matrix: ATO プロセスの合理化
Azure Blueprint プログラムでは、お客様の機関またはシステムに固有の運用承認 (ATO) プロセスを支援するためのドキュメントに、お客さまがアクセスできます。 この記事では、Azure Blueprint Customer Responsibilities Matrix (CRM) と、セキュリティ担当者によるこのドキュメントの活用方法について詳しく説明します。

Azure Blueprint CRM は実用目的で設計されています。 制御要件のみを定義しているスプレッドシート形式のドキュメントであり、Azure でシステム構築を行うお客様は、この制御要件を満たすセキュリティ制御を実装する必要があります。

> [!NOTE]
> Azure Blueprint では、"お客様" とは Azure IaaS/PaaS 環境内で直接システム構築を行う組織を指します。 Azure のお客様には、Azure を直接のシステム構築基盤とする政府または政府機関に代わってシステム構築を行う、サードパーティ ISV も含まれます。
> 
> 

CRM では、<a href="http://nvlpubs.nist.gov/nistpubs/SpecialPublications/NIST.SP.800-53r4.pdf">NIST SP 800-53 Revision 4 </a>の制御について明示していますが、これにはお客様の責任も含まれます。 この制御には、お客様が全責任を担う制御と、お客様と Azure の間で責任を分担する制御があります。 各制御は制御副要件まで細かく分類され、可能な限り明確に示されています。

たとえば、ATO を必要とするシステム向けに、AC-1 ではアクセス制御ポリシーおよび手続きの文書化を求めています。 この制御について、Azure では、Azure のインフラストラクチャとプラットフォームの管理に使用されるアクセス制御メカニズムに関する内部ポリシーおよび手続きを用意しています。 お客様は、Azure で構築した特定のシステム内で使用される独自のアクセス制御ポリシーおよび手続きを作成する必要もあります。 CRM には、特定のコンテンツを含むためのポリシーと手続きを必要とする AC-1a、このようなドキュメントを年単位で確認および更新することをお客様に求める AC-1b という制御部分が示されています。 

CRM と Azure Blueprint System Security Plan Template へのアクセス方法については、AzureBlueprint@microsoft.com までメールでお問い合わせください。

CRM についてのフィードバックは、お客様の ATO プロセスを最大限に合理化できるようドキュメントの改訂を行うために極めて重要ですので、ぜひお寄せください。
