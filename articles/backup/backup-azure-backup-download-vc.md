# コンテナー資格情報を使用して、Azure Backup サービスを使用して認証するには

(Windows クライアントまたはサーバーの Windows Server または SCDPM) は、内部設置型サーバーは、Azure へのデータをバックアップできます前に、バックアップ コンテナーに認証する必要があります。認証は、「コンテナー資格情報」を使用して実現されます。コンテナー資格情報の概念は、Azure PowerShell で使用されている「発行の設定」のファイルの概念に似ています。

## 資格情報のコンテナー ファイルとは

コンテナーの資格情報ファイルには、各バックアップ コンテナー用のポータルによって生成される証明書です。ポータルの Access Control サービス (または ACS) を公開キーをアップロードします。証明書の秘密キーは、マシンの登録のワークフローの入力として指定したワークフローの一部としてユーザーには利用できます。これは、Azure Backup サービスで識別される、資格情報のコンテナーにバックアップ データを送信する、コンピューターを認証します。

これは、登録のワークフローの中にのみ、資格情報コンテナー資格情報を使用するおく必要があります。これは、コンテナー資格情報ファイルが損なわれることを確認するにはユーザーの責任です。すべての悪意のあるユーザーの手になるには場合、同じ資格情報コンテナーに対して、その他のマシンを登録するコンテナーの資格情報ファイルを使用できます。ただし、ように、バックアップ データは暗号化するには、顧客に属している、パスフレーズを使用して、既存のバックアップ データが侵害されることはできません。この問題を緩和するのには、コンテナー資格情報の設定は 48hrs で期限切れにします。何回でも – バックアップ コンテナーのコンテナー資格情報をダウンロードできますが、最新の資格情報コンテナー資格情報ファイルのみが登録ワークフローの間に該当します。

## 資格情報コンテナーの資格情報ファイルをダウンロードします。

資格情報コンテナーの資格情報ファイルは、Azure ポータルから、セキュリティで保護されたチャネルを介してダウンロードされます。Azure Backup サービスは、証明書の秘密キーを認識しないと、ポータルまたはサービスでは、秘密キーは保持されません。ローカル コンピューターに資格情報コンテナー資格情報をダウンロードするには、次の手順を使用します。

1.  サインイン、 [管理ポータル](https://manage.windowsazure.com/)
2.  クリックして **復旧サービス** 、左側のナビゲーション ウィンドウで、バックアップ コンテナーを作成する] を選択します。クイック スタートのビューに、バックアップ コンテナーを取得するには、クラウド アイコンをクリックします。 <br/> ![簡単に表示][1]

3.  [クイック スタート] ページで、次のようにクリックします。 **コンテナー資格情報のダウンロード**です。ポータルは、ダウンロードの使用可能な資格情報コンテナーの資格情報のファイルを生成します。 <br/> ![ダウンロード][2]

4.  ポータルは、資格情報コンテナーの名前と現在の日付の組み合わせを使用して、資格情報コンテナーの資格情報を生成します。クリックして **保存** をローカル アカウントのダウンロード フォルダーに、コンテナー資格情報をダウンロードまたは資格情報コンテナー資格情報の場所を指定するには、[保存] メニューの [から名前を付けて保存を選択します。

## 注
+ 年 3 月の 2015年の時点でユーザーがないプログラム (例: PowerShell) のコンテナー資格情報をダウンロードする方法です。

+ 資格情報コンテナー資格情報をコンピューターからアクセスできる場所に保存することを確認します。ファイル共有と SMB に格納されている場合は、アクセス許可を確認します。

+ コンテナーの資格情報ファイルは、登録のワークフローの中にのみ使用されます。

+ コンテナーの資格情報ファイル 48hrs 後に有効期限が切れるし、ポータルからダウンロードできます。

+ Azure Backup を参照してください [よく寄せられる質問](backup-azure-backup-faq.md) 、ワークフローの質問などです。
