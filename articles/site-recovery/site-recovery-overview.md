#  Site Recovery の概要

Site Recovery サービスは、Azure またはオンプレミスのセカンダリ データセンターへのレプリケーションとフェールオーバーを調整および自動化することにより、オンプレミスの物理サーバーおよび仮想マシンを保護する、堅牢なビジネス継続性と障害復旧 (BCDR) ソリューションを構築するのに役立ちます。

- **簡素化** - Site Recovery は、レプリケーションの構成を容易にすることで、BCDR 戦略を簡素化し、オンプレミスのワークロードとアプリケーションに対するフェールオーバーと復旧を実行するのに役立ちます。
- **レプリケーション** - Azure ストレージ、またはセカンダリ データセンターに、オンプレミスのワークロードをレプリケートできます。 
- **コンテナー** - レプリケーションを管理するために、選択する Azure リージョンに、Site Recovery コンテナーを設定します。すべてのメタデータは、そのリージョン内に留まります。
- **メタデータ** - アプリケーションのデータは Azure に送信されません。レプリケーションとフェールオーバーを調整するために Site Recovery に必要なのは、仮想マシン名と VMM クラウド名などのメタデータのみです。 
- **Azure への接続** - デプロイ シナリオによっては、管理サーバーが Azure と通信します。たとえば、オンプレミスの VMM クラウド内に配置されている仮想マシンをレプリケートする場合、VMM サーバーは、暗号化された送信 HTTPS 接続を介して Site Recovery と通信します。仮想マシンまたは Hyper-V ホストからの接続は必要ありません。
- **Hyper-V レプリカ** - Azure Site Recovery は、レプリケーション プロセスに Hyper-V レプリカを利用します。また、2 つのオンプレミスの VMM サイト間でレプリケートする場合は、SAN レプリケーションも使用できます。Site Recovery は、最初のレプリケーションに VHD 全体ではなくデータのブロックのみをレプリケートするスマート レプリケーションを使用します。進行中のレプリケーションでは、差分の変更のみがレプリケートされます。Site Recovery は、オフライン データ転送をサポートし、WAN オプティマイザーと連携します。
- **価格** - Site Recovery の料金についての[詳細](pricing/details/site-recovery)を参照できます。
