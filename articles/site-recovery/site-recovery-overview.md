#  Site Recovery の概要

Site Recovery サービスは、Azure またはオンプレミスのセカンダリ データセンターへのレプリケーションとフェールオーバーを調整および自動化することにより、オンプレミスの物理サーバーおよび仮想マシンを保護する、堅牢なビジネス継続性と障害復旧 (BCDR) ソリューションを構築するのに役立ちます。

- **簡素化** - Site Recovery は、レプリケーションの構成を容易にすることで、BCDR 戦略を簡素化し、オンプレミスのワークロードとアプリケーションに対するフェールオーバーと復旧を実行するのに役立ちます。
- **レプリケーション** - Azure ストレージ、またはセカンダリ データセンターに、オンプレミスのワークロードをレプリケートできます。 
- **コンテナー** - レプリケーションを管理するために、選択する Azure リージョンに、Site Recovery コンテナーを設定します。すべてのメタデータは、そのリージョン内に留まります。
- **メタデータ** - アプリケーションのデータは Azure に送信されません。レプリケーションとフェールオーバーを調整するために Site Recovery に必要なのは、仮想マシン名と VMM クラウド名などのメタデータのみです。 
- **Azure への接続** - デプロイ シナリオによっては、管理サーバーが Azure と通信します。たとえば、オンプレミスの VMM クラウド内に配置されている仮想マシンをレプリケートする場合、VMM サーバーは、暗号化された送信 HTTPS 接続を介して Site Recovery と通信します。仮想マシンまたは Hyper-V ホストからの接続は必要ありません。
- **Hyper-V レプリカ** - Azure Site Recovery は、レプリケーション プロセスに Hyper-V レプリカを利用します。また、2 つのオンプレミスの VMM サイト間でレプリケートする場合は、SAN レプリケーションも使用できます。Site Recovery は、最初のレプリケーションに VHD 全体ではなくデータのブロックのみをレプリケートするスマート レプリケーションを使用します。進行中のレプリケーションでは、差分の変更のみがレプリケートされます。Site Recovery は、オフライン データ転送をサポートし、WAN オプティマイザーと連携します。
- **価格** - Site Recovery の料金についての[詳細](pricing/details/site-recovery)を参照できます。


## デプロイ シナリオ

次の表は、Site Recovery でサポートされているレプリケーションのシナリオをまとめたものです。

**レプリケート先** | **レプリケート元 (オンプレミス)** | **詳細** | **記事**
---|---|---|---
Azure | Hyper-V サイト | Hyper-V サイトとして定義した 1 つまたは複数のオンプレミスの Hyper-V ホスト サーバーに存在する仮想マシンを Azure にレプリケートします。VMM サーバーは必要ありません。 | 詳細については、[こちら](site-recovery-hyper-v-site-to-azure.md)を参照してください。
Azure| VMM サーバー | VMM クラウド内に配置した 1 つまたは複数のオンプレミスの Hyper-V ホスト サーバーに存在する仮想マシンを Azure にレプリケートします。 | 詳細については、[こちら](site-recovery-vmm-to-azure.md)を参照してください。 
Azure | 物理 Windows サーバー | 物理 Windows サーバーまたは物理 Linux サーバーを Azure にレプリケートします。 | 詳細については、[こちら](site-recovery-vmware-to-azure.md)を参照してください。
Azure | VMware 仮想マシン | VMWare 仮想マシンを Azure にレプリケートします。 | 詳細については、[こちら](site-recovery-vmware-to-azure.md)を参照してください。
セカンダリ データセンター | VMM サーバー | VMM クラウド内に配置したオンプレミスの Hyper-V ホスト サーバーに存在する仮想マシンを、別のデータセンター内のセカンダリ VMM サーバーにレプリケートします。 | 詳細については、[こちら](site-recovery-vmm-to-vmm.md)を参照してください。
セカンダリ データセンター | SAN を使用した VMM サーバー | SAN レプリケーションを使用して、VMM クラウド内に配置したオンプレミスの Hyper-V ホスト サーバーに存在する仮想マシンを、別のデータセンター内のセカンダリ VMM サーバーにレプリケートします。| 詳細については、[こちら](site-recovery-vmm-san.md)を参照してください。
セカンダリ データセンター | 単一の VMM サーバー | VMM クラウド内に配置したオンプレミスの Hyper-V ホスト サーバーに存在する仮想マシンを、同じ VMM サーバーのセカンダリ クラウドにレプリケートします。 | 詳細については、[こちら](site-recovery-single-vmm.md)を参照してください。
