# Azure AD の使用開始方法

[Azure AD とは](active-directory-whatis.md)<br> [Azure AD の機能](active-directory-works.md)<br> [次のステップ](active-directory-next-steps.md)<br> [詳細](active-directory-learn-map.md)


使い始める準備はできていますか。 Azure AD は誰でも無料で使用できます。今すぐ Azure 評価版サブスクリプションにサインアップして、ディレクトリを使用できます。購入するかどうかにかかわらず、ディレクトリはお客様のものです。[今すぐ Azure AD をお試しください](http://azure.microsoft.com/trial/get-started-active-directory/)

## テスト環境、ステージング環境、運用環境に使用するディレクトリをどのようにして決めればよいでしょうか。

Office 365 などの Microsoft クラウド サービスのサブスクリプションを既にお持ちの場合、Azure AD には既にディレクトリがあるので、それを使用できます。Office 365 にサインインするユーザーは、そのディレクトリによって既に認証されています。Office 365 へのサインインに使用している会社または学校のアカウントを使用して、Azure にサインアップすることだけが必要です。その後は、Azure 管理ポータルでディレクトリを管理し、必要に応じてテストなどのためにディレクトリを追加できます。[Azure で Office 365 サブスクリプションのディレクトリを管理する方法の詳細](active-directory-how-subscriptions-associated-directory.md#manage-the-directory-for-your-office-365-subscription-in-azure)。

Microsoft クラウド サービスのサブスクリプションをまだ持っていない場合は、Azure にサインアップすると会社や学校用に新しいディレクトリがプロビジョニングされ、プライマリ ディレクトリとしてそれを使用できます。


## 登録済みのドメイン名を追加する方法
カスタム ドメイン名を追加したり、登録済みのドメイン名を使用したりできます。 [カスタム ドメインの追加についての詳細](active-directory-add-domain.md)。

## 会社のブランドを追加する方法
Azure AD Premium では、会社のロゴやその他のブランドを、Azure のサインイン ページおよびアクセス パネル ページに追加できます。[サインイン ページとアクセス パネル ページに対する会社のブランドの追加](active-directory-add-company-branding.md)

## Azure AD を別のディレクトリと統合する方法
パスワードの同期など、オンプレミスのディレクトリと Azure AD の間のディレクトリ同期を設定し、オンプレミスおよびクラウドのアプリケーションへのアクセスに対してシングル サインオンを有効にできます。[ディレクトリの統合についての詳細](active-directory-aadconnect.md)。

## ユーザーを作成および編集する方法
Microsoft クラウド サービスにアクセスするユーザーごとにアカウントを作成します。既定では、ユーザーには管理者のアクセス許可はありませんが割り当てることができます。[Azure AD ユーザーの作成についての詳細](active-directory-create-users.md)。

## アクセスおよび使用状況レポートの表示
Azure AD には、管理者用に多くのレポートがあります。Azure AD の有料エディションではさらに多くのレポートを使用でき、カスタム レポートを作成するためのレポート API もあります。[Azure AD のレポートについての詳細](active-directory-view-access-usage-reports.md)。
