# <a name="building-secure-workstations"></a>セキュリティで保護されたワークステーションの構築

セキュリティで保護された分離したワークステーションは、重要なサービスの管理者、開発者、オペレーターのような機密性の高い役割のセキュリティには非常に重要です。 その他の多くのセキュリティ制御や保証は、基礎となるクライアント ワークステーションのセキュリティが侵害された場合、失敗するか効果がありません。

このドキュメントでは、最初のセキュリティ制御を設定する方法を含むステップ バイ ステップの詳細な手順で、セキュリティで保護されたクライアント ワークステーションを構築する方法について説明します。 この種類のワークステーションは、このリファレンスを使用して基盤としている、特権アクセスを提供するワークステーション (PAW) と呼ばれています。 ただし、このガイダンスでは、サービスを管理するクラウド ベースのテクノロジに注目し、Windows 10RS5、Microsoft Defender ATP、Azure Active Directory、および Intune で導入されたセキュリティ機能を紹介します。

## <a name="why-securing-workstation-access-is-important"></a>セキュリティで保護されたワークステーションのアクセスが重要な理由

クラウド サービスの急速な導入と、どこからでも作業できるようになったことで、新たな悪用の方法が生み出されました。 攻撃者は、管理者が操作して特権付きリソースへのアクセス権を付与できるデバイスに対する、脆弱なセキュリティ制御を悪用しています。

[Verizon 脅威レポート](https://enterprise.verizon.com/resources/reports/dbir/)、および[セキュリティ インテリジェンス レポート](https://aka.ms/sir)の特権の悪用に記載されているように、組織を侵害するために使用される上位 5 つのメカニズムの中にサプライ チェーン攻撃があり、2018 年に報告されたインシデントで 2 番目に最も多く検出された戦術です。

ほとんどの攻撃者は以下の経路をたどります。

* 入口を見つけるために、多くの場合は業界を特定して、偵察から始める
* 収集された情報を分析して、価値が低いと思われるワークステーションにアクセス (侵入) するための最高の手段を特定する
* とどまって、[水平](https://en.wikipedia.org/wiki/Network_Lateral_Movement)移動する手段を検討する
* 機密データを抜き取る

攻撃者は、リスクが低いと思われるデバイスや過小評価されていると思われるデバイスに、偵察のために頻繁に侵入します。 これらの脆弱性のあるデバイスは、水平移動の機会を見つけ、管理ユーザーとデバイスを探し、価値の高いデータを特定するために使用され、特権ユーザーのロールを得た後で情報の抜き取りに成功するために使用されます。

![一般的なセキュリティ侵害のパターン](./media/concept-azure-managed-workstation/typical-timeline.png)

このドキュメントでは、価値の低い生産性向上デバイスからの水平移動や攻撃から守るために管理とサービスを分離することでコンピューティング デバイスを保護するソリューションについて説明します。 この設計では、機密性の高いクラウド リソースの管理やアクセスに使用されるデバイスに侵入される前に連鎖を断ち切ることで、侵害の達成に成功する可能性を減らすことができます。 説明するソリューションでは、次のような、Microsoft 365 Enterprise スタックの一部であるネイティブの Azure サービスを利用します。

* デバイス管理用の Intune (アプリケーションと URL のホワイトリストを含む)
* Autopilot (デバイスの設定とデプロイ、および更新用) 
* Azure AD (ユーザー管理、条件付きアクセス、および多要素認証用)
* Windows 10 (現在のバージョン) (デバイス正常性構成証明書とユーザー エクスペリエンス用)
* Microsoft Defender Advanced Threat Protection (ATP) (クラウド管理によるエンドポイントの保護、検出、および応答用)
* Azure AD PIM (リソースへの Just-In-Time (JIT) の特権アクセスなどの認証の管理用)

## <a name="who-benefit-from-using-a-secure-workstation"></a>セキュリティで保護されたワークステーションを使用するメリットがある人

すべてのユーザー、およびオペレーターには、セキュリティで保護されたワークステーションを使用するメリットがあります。 PC やデバイスを侵害する攻撃者は、キャッシュされたすべてのアカウントを偽装したり、資格情報を使用したり、ログオンしているときにそのデバイスで使用されるトークンを使用したりといった、さまざまなことを実行する可能性があります。 特権アカウントが使用されているデバイスは水平移動や特権エスカレーション攻撃の標的となるため、このリスクにより、管理者権限を含むすべての特権ロールで使用されるデバイスをセキュリティ保護することが非常に重要になります。 これらのアカウントは、次のようなさまざまな資産に使用されている場合があります。

* オンプレミスとクラウドベースのシステムの管理者
* 重要なシステムの開発者のワークステーション
* 露出度の高いソーシャル メディア アカウントの管理者
* SWIFT 支払い端末のような機密性の高いワークステーション
* 企業機密を扱うワークステーション

Microsoft では、リスク軽減のためにこれらのアカウントが使用されている特権ワークステーションに対して、管理者特権でのセキュリティ制御を実装することをお勧めします。 追加のガイダンスについては、「[Azure Active Directory 機能のデプロイ ガイド](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-deployment-checklist-p2)」、[Office 365 ロードマップ](https://aka.ms/o365secroadmap)、および[特権アクセスのセキュリティ保護のロードマップ](https://aka.ms/sparoadmap)に関するページを参照してください。

## <a name="why-dedicated-workstations"></a>専用のワークステーションである理由

既存のデバイスにセキュリティを追加することはできますが、安全な基盤から始めることをお勧めします。 既知の正常なデバイスと一連の既知のセキュリティ制御から始めることで、組織では、より高いレベルのセキュリティを適切に維持することができます。 何気ないメールや Web の閲覧によって許容される攻撃ベクトルの数が増え続けている状況で、デバイスを信頼できると保証することはますます困難になっています。 このガイドは、標準的な生産性、閲覧、およびメールのタスクから分離された専用のワークステーションが完成しているという前提の下で役に立ちます。 デバイスから生産性、Web の閲覧、およびメールを取り除くと、生産性に悪影響を及ぼす可能性がありますが、この保護は通常、ジョブ タスクで明示的には要求せず、セキュリティ インシデントのリスクが高いシナリオに適しています。

> [!NOTE]
> ここでの Web の閲覧は、任意の Web サイトへの通常のアクセスを指しています。これは高いリスクであり、Web ブラウザーを使用してサービス (Azure、Office 365、その他のクラウド プロバイダー、SaaS アプリケーションなど) のために少数の既知の正常な管理用 Web サイトにアクセスすることとは明らかに違います。

コンテインメント戦略では、敵対者が機密性の高い資産にアクセスするために克服する必要のある制御の数と種類を増やすことで、セキュリティの向上を保証します。 ここで開発したモデルは、階層化された特権モデルを使用して特定のデバイスへの管理特権の包含を実現します。

## <a name="supply-chain-management"></a>サプライ チェーン管理

セキュリティで保護されたワークステーションに不可欠なのは、使用するワークステーションが信頼されている、つまり「信頼のルート」であるサプライ チェーン ソリューションです。 このソリューションは、[Microsoft Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot) テクノロジを使用して信頼のルートに対応します。 セキュリティで保護されたワークステーションでは、Microsoft Autopilot は、製造元から既知の正常な状態を提供する Microsoft OEM 用に最適化された Windows 10 デバイスを活用する機能を提供します。 Microsoft Autopilot では、信頼できない可能性があるデバイスを再イメージ化する代わりに、設定とポリシーを適用し、アプリをインストールし、使用中の Windows 10 のエディションの変更 (たとえば、高度な機能をサポートするための Windows 10 Pro から Windows 10 Enterprise への変更) も行って、Windows デバイスを “ビジネスに即応する” 状態に変換することができます。

![セキュリティで保護されたワークステーションのレベル](./media/concept-azure-managed-workstation/supplychain.png)

## <a name="device-roles-and-profiles"></a>デバイスのロールとプロファイル

このガイダンスの全体を通して、ユーザー、開発者、および IT 運用スタッフのためのより安全なソリューションを実現する、さまざまなセキュリティ プロファイルとロールが取り上げられています。 これらのプロファイルは、使いやすさとリスクのバランスを取りながら、強化された、またはセキュリティで保護されたワークステーションから利点を得られる組織の一般的なユーザーをサポートするように調整されています。 ガイダンスでは、業界で広く認められている標準に基づいた設定の構成を提供します。 このガイダンスを使用して、Windows 10 を強化し、セキュリティ機能とリスクの管理に役立つポリシーとテクノロジを使用してデバイスまたはユーザーの侵害に関連するリスクを減らす方法を説明します。
![セキュリティで保護されたワークステーションのレベル](./media/concept-azure-managed-workstation/seccon-levels.png)

* **低セキュリティ** – 管理対象の標準的なワークステーションは、ほとんどの自宅や小規模企業で手始めとして使用するのに適しています。 これらのデバイスは Azure AD 登録済みで Intune で管理されています。 このプロファイルは、ユーザーにアプリケーションの実行と Web サイトの閲覧を許可します。 [Microsoft Defender](https://www.microsoft.com/windows/comprehensive-security) などのマルウェア対策ソリューションを有効にする必要があります。
* **強化されたセキュリティ** – エントリ レベルの保護されたソリューションであり、ホーム ユーザー、小規模企業のユーザー、および一般的な開発者に適しています。
   * 強化されたワークステーションは、低セキュリティのプロファイルのセキュリティを強化するための、ポリシー ベースの手段を提供します。 このプロファイルにより、安全な手段で、顧客データを処理したり、メールのチェックや Web の閲覧などの生産性向上ツールを使用できるようにしたりすることができます。 強化されたワークステーションを使用すると、監査ポリシーを有効にし、Intune にログを記録することで、ユーザーの動作や、ワークステーションのプロファイルの使用を監査することができます。 このプロファイルでは、ワークステーションは、このコンテンツで説明し、Enhanced Workstation - Windows10 (1809) スクリプトでデプロイされたセキュリティ制御とポリシーを有効にします。 このデプロイでは、[Advanced Threat Protection (ATP)](https://docs.microsoft.com/office365/securitycompliance/office-365-atp) を使用した高度なマルウェア保護も活用します
* **高セキュリティ** – ワークステーションの攻撃対象領域を減らすための最も効果的な手段は、ワークステーションの自己管理機能を削除することです。 ローカル管理者権限を削除することは、セキュリティを強化する 1 つの手段であり、正しく実装されていない場合は生産性に影響を与える可能性があります。 高セキュリティのプロファイルは、強化されたセキュリティのプロファイルを基盤として、ローカル管理の削除という 1 つの重要な変更を加えています。このプロファイルは、役員などの要人であるユーザーや、給与計算 (つまり勤務の承認) やプロセスなどの機密データに接する可能性があるユーザーを支援するように設計されています。
   * 高セキュリティのユーザー プロファイルでは、使い勝手の良さを維持しながらメールや Web の閲覧などの生産性向上アクティビティを引き続き実行できるようにする、より高度に制御された環境が求められます。 ユーザーは、Cookie、お気に入り、操作に使用できるその他のショートカットなどの機能を期待しています。 ただし、このようなユーザーは変更を加えたりデバイスをデバッグしたりする必要がない場合があり、ドライバーをインストールする必要はありません。 高セキュリティのプロファイルは、High Security - Windows10 (1809) スクリプトを使用してデプロイされます。
* **特殊** – 開発者と IT 管理者は、その役割で攻撃者に関心のあるシステムを変更できるため、攻撃者にとっては魅力的な標的です。 特殊ワークステーションは、高セキュリティのワークステーションにデプロイされた取り組みを必要とし、さらに、ローカル アプリケーションの管理、インターネット Web サイトの制限、および ActiveX、Java、ブラウザー プラグイン、Windows デバイス上のその他の既知の高リスクの制御などのリスクの高い生産性機能を制限することで、そのセキュリティを強化します。 このプロファイルでは、ワークステーションは、このコンテンツで説明し、DeviceConfiguration_NCSC - Windows10 (1803) SecurityBaseline スクリプトでデプロイされたセキュリティ制御とポリシーを有効にします。
* **セキュリティ保護** – 管理者アカウントを侵害できる攻撃者は、通常、データ盗難、データ改ざん、またはサービスの中断によってビジネスに重大な損害を与える可能性があります。 この強化された状態では、ワークステーションは、ローカル アプリケーション管理の直接的な制御を制限するすべてのセキュリティ制御とポリシーを有効にし、生産性向上ツールは削除されます。 その結果、フィッシング攻撃が成功する最も一般的な方法であるメールやソーシャル メディアがブロックされるため、デバイスの侵害はより難しくなります。  セキュリティ保護されたワークステーションは、Secure Workstation - Windows 10 (1809) SecurityBaseline スクリプトを使用してデプロイできます。

   ![セキュリティ保護されたワークステーション](./media/concept-azure-managed-workstation/secure-workstation.png)

   セキュリティ保護されたワークステーションは、管理者に、明確なアプリケーション制御とアプリケーション保護を備えた、強化されたワークステーションを提供します。 このワークステーションは、資格情報、デバイス、および悪用からの保護を使用して、悪質な行動からホストを保護します。 さらに、すべてのローカル ディスクは BitLocker 暗号化を使用して暗号化されます。

* **分離** – このカスタムのオフラインのシナリオは、このスペクトルの一番端にあります (このケースのインストール スクリプトは提供されていません)。 組織では、サポートされていない、またはパッチが適用されていない従来のオペレーティング システムを必要とする高価な生産ラインや生命維持装置など、分離されたビジネスに不可欠な機能を管理する必要がある場合があります。 セキュリティは非常に重要であり、クラウド サービスは利用できないため、組織では、これらのコンピューターを手動で管理/更新するか、分離された Active Directory フォレスト アーキテクチャ (Enhanced Security Admin Environment (ESAE) など) を使用してそれらを管理します。 このような状況では、基本の Intune 以外のすべてのアクセス権の削除と ATP 正常性チェックを考慮する必要があります。
   * [Intune ネットワーク通信の要件](https://docs.microsoft.com/intune/network-bandwidth-use)
   * [ATP ネットワーク通信の要件](https://docs.microsoft.com/azure-advanced-threat-protection/configure-proxy)
