#アプリケーション プロキシによるハイブリッド アクセスの有効化
Microsoft Azure AD アプリケーション プロキシを使用して、プライベート ネットワーク内で実行されるアプリケーションを発行することができます。これにより、ネットワークの外部から、あらゆるデバイス上のアプリケーションに安全にアクセスすることができます。Azure AD アプリケーション プロキシを使用することで、プライベート オンプレミスまたはクラウド ネットワーク内でアプリケーションを発行できます。Azure AD 内にすぐに使用できるサービスとして構成でき、ユーザーに必要な操作は使用環境内のサーバーにアプリケーション プロキシ コネクタをインストールするだけです。

##動作のしくみ

次の手順では、ハイブリッド環境でアプリケーションをどのように発行できるかを一覧表示します。
 
1.	コネクタは企業ネットワークにデプロイされます。(冗長性とスケールのために複数のコネクタをデプロイすることができます)。
2.	コネクタはクラウド サービスに自動接続されます。
3.	ユーザーは、コネクタ経由でリソースにトラフィックをルーティングするクラウド サービスに接続します。

次の図に、アプリケーション プロキシの動作のしくみを詳しく示します。

 ![Azure AD アプリケーション プロキシの動作のしくみ](./media/active-directory-appssoaccess-whatis/azureappproxxy.png)

 
1.	ユーザーがアプリケーション プロキシ経由でアプリケーションにアクセスすると、認証のための Azure AD ログオン ページが表示されます。
2.	ログオンに成功すると、トークンが生成され、ユーザーに送信されます。
3.	ユーザーはここでアプリケーション プロキシにトークンを送信します。アプリケーション プロキシは、そのトークンからユーザー プリンシパル名 (UPN) とセキュリティ プリンシパル名 (SPN) を取得し、コネクタに要求を送信します。
4.	コネクタは、ユーザーの代理として内部 (Windows) 認証に使用できる Kerberos チケットを要求します。(Kerberos の制約付き委任)
5.	 Kerberos チケットは Active Directory から取得されます。
6.	取得されたチケットは検証場所のアプリケーション サーバーに送信されます。
7.	 応答は、アプリケーション プロキシ経由でエンドユーザーに送信されます。Web アプリケーションを発行する場合の要件が 1 つだけあります。それは、プライベート ネットワークの Web アプリケーションに、コネクタがインストールされているサーバーからアクセスできる必要があるということです。つまり、コネクタはアプリケーション サーバー自体にも、環境内の他のサーバーにもインストールすることができます。コネクタが Web アプリケーションにアクセス可能であれば何も問題はありません。

##Azure AD アプリケーション プロキシ経由でオンプレミス アプリケーションを統合する方法
Azure AD のアクセス パネルでは、Azure AD にオンプレミス アプリケーションを発行することができます。従う必要がある手順は以下のとおりです。

1. Azure AD でアプリケーション プロキシを有効にし、コネクタをインストールして登録します。詳細な手順については、「[Azure AD アプリケーション プロキシの有効化](active-directory-application-proxy-enable/#step-1-enable-application-proxy-in-azure-ad.md)」を参照してください。
2. Azure AD アプリケーション プロキシを使用してアプリケーションを発行します。詳細な手順については、「[Azure AD アプリケーション プロキシを使用してアプリケーションを発行する](active-directory-application-proxy-publish.md)」を参照してください。
