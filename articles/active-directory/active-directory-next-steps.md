# Azure AD を使用する次のステップ


###このトピックに関するその他の記事

## Azure AD でユーザーを管理する
ユーザー アカウントの管理は、ユーザーを作成または削除したり、アクセス許可を割り当てるだけではありません。アクセスおよび使用状況に関するレポートの生成、パスワード ポリシーの設定、パスワードのセルフ サービス リセットの有効化、多要素認証の設定、その他を行うことができます。[ユーザー管理の詳細](active-directory-create-users.md)。

## パスワードの同期と管理
ユーザーがオンプレミス ネットワークへのサインインに使用するときと同じパスワードを使用して Microsoft クラウド サービス (Office 365、Dynamics CRM、Microsoft Intune など) にサインインできるように、ユーザーのパスワードを同期します。[パスワード管理の詳細](active-directory-manage-passwords.md)。

## Azure AD でのグループの管理
アクセス許可と権限の共通セットの割り当てを、アカウントごとではなく、グループを使用して、多数のアカウントに対して一度に行います。ローカルなオンプレミスの Windows Server Active Directory からグループを同期することもできます。Azure Active Directory でのグループ管理の詳細について学習してください。[グループ管理の詳細](active-directory-manage-groups.md)。

## 登録済みのモバイル デバイスに条件付きアクセス ポリシーを適用します。
Azure AD のデバイス登録は、ユーザーのサインイン時にデバイスの認証に使用される ID でデバイスをプロビジョニングします。認証済みのデバイスおよびデバイスの属性を使用して、クラウドおよびオンプレミスでホストされるアプリケーションに条件付きアクセス ポリシーを適用できます。[Azure AD デバイス登録の詳細](active-directory-conditional-access.md)。
