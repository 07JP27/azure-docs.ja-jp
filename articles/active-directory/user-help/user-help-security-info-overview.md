# <a name="security-info-preview-overview"></a>セキュリティ情報 (プレビュー) の概要

**[セキュリティ情報]** ページでは、2 段階認証の方法とパスワード リセット情報を設定できます。 また、個人のセキュリティ情報と既定の認証方法を更新することもできます。

[!INCLUDE [preview-notice](../../../includes/active-directory-end-user-preview-notice-security-info.md)]

>[!Important]
>このコンテンツはユーザーを対象としています。 管理者の方は、Azure Active Directory (Azure AD) 環境の設定と管理方法の詳細については、「[Azure Active Directory のドキュメント](https://docs.microsoft.com/azure/active-directory)」を参照してください。

## <a name="security-info-articles"></a>セキュリティ情報に関する記事

セキュリティ情報の追加、更新、削除に関する詳細と手順については、(使用する方法に応じて) 次の記事を参照してください。

|記事 |説明 |
|------|------------|
|[サインイン ページのプロンプトからセキュリティ メソッドを追加する](security-info-setup-signin.md)| サインイン ページに表示されたプロンプトからセキュリティ情報を設定する方法について説明します。 このプロンプトは、組織から要求されたセキュリティ情報をまだ設定していない場合にのみ表示されます。 プロンプトが表示されない場合でも、この表の他の記事の説明に従って認証方法の変更や追加を行うことができます。|
|[Microsoft Authenticator アプリを認証方法として追加する](security-info-setup-auth-app.md)| Microsoft Authenticator アプリを認証方法として設定する方法について説明します。|
|[テキスト メッセージを認証方法として追加する](security-info-setup-text-msg.md)| テキスト メッセージを認証方法として設定する方法について説明します。|
|[電話を認証方法として追加する](security-info-setup-phone-number.md)| 電話番号を認証方法として設定する方法について説明します。|
|[パスワードをリセットするためのメール アドレスを追加する](security-info-setup-email.md)| パスワードをリセットする際に使用するメール アドレスを追加する方法について説明します。|
|[パスワードをリセットするためのセキュリティの質問を追加する](security-info-setup-questions.md)| パスワードをリセットできるようにするために、事前定義のセキュリティの質問を追加する方法について説明します。|
|[2 段階認証をサポートしないアプリのアプリ パスワードを管理する](security-info-app-passwords.md)| 2 段階認証を使用する組織で、2 段階認証をサポートしないアプリのアプリ パスワードを作成する方法について説明します。|
|[サインイン方法](user-help-sign-in.md)|指定した確認方法を使用してサインインする方法について説明します。|
