# <a name="workday-and-azure-active-directory"></a>Workday と Azure Active Directory

事前構築済みの統合を使用して Workday を Azure Active Directory (Azure AD) に接続し、従業員の入社、異動、退職時にユーザー アカウントを自動的にプロビジョニングまたはプロビジョニング解除します。

![Workday アプリのロゴの表示](./media/workday.png)

> [!div class="nextstepaction"]
> [Azure AD を使用して Workday を試す >](https://aka.ms/WorkdayTryItNow)

## <a name="what-can-you-do-with-workday-and-azure-ad"></a>Workday と Azure AD で行えること

- **ユーザー プロビジョニングの自動化** - Workday でグループ、ロール、またはその他の属性に基づいて、従業員にアプリケーションを自動的に割り当てます。
- **従業員プロファイルの同期** - Azure AD と Workday の間で従業員のデータが最新の状態に保たれるようにします。
- **オンボードとオフボードの簡易化** - 新規採用者が初日から生産的に作業できるようにそのユーザー アカウントを自動的にプロビジョニングし、従業員が退職したときにそのユーザー アカウントを自動的にプロビジョニング解除することでセキュリティ上のリスクを軽減します。
- **簡単なシングル サインオン** - ユーザー名とパスワードを排除し、Azure AD でホストされている組織アカウントを使用してユーザーがサインインできるようにします。

## <a name="get-started"></a>作業開始

簡単な構成のガイドを使用して、Workday の[ユーザー プロビジョニング](https://aka.ms/WorkdayProvisioning)と[シングル サインオン](https://aka.ms/WorkdaySSODoc)を設定します。
