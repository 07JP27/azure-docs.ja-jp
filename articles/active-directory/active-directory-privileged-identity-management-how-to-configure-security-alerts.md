# Azure AD Privileged Identity Management でセキュリティ アラートを構成する方法

## セキュリティのアラート
Azure Privileged Identity Management (PIM) では、次のアラートが生成されます。アラートは、PIM ダッシュボードの [アラート] セクションで表示できます。

| アラート: | トリガー | 推奨 |
| ----- | ------- | -------------- |
| **Permanent activation** (永続的にアクティブ化されています) | 管理者が PIM の外部でロールに永続的に割り当てられました。 | 新しいロールの割り当てを確認し、必要に応じて一時的な割り当てに変更してください。 |
| **特権付きロールのアクティブ化の疑わしい更新です** | 設定で許可された時間に行われた同じロールの再アクティブ化が多すぎます。 | ユーザーに連絡して、ロールを正常にアクティブ化できることを確認してください。 |
| **ロールのアクティブ化に弱い認証が構成されています** | MFA が有効ではないロールが設定に存在します。 | すべてのロールのアクティブ化に対して MFA を要求することを検討してください。 |
| **Redundant administrators** (必要以上に管理者が指定されています) | 最近ロールをアクティブ化していない一時的な管理者がいます。 | 不要になったロールの割り当てを削除してください。 |
| **Too many global administrators** (グローバル管理者が多すぎます) | 推奨されている数よりも多くのグローバル管理者がいます。 | 不要になったロールの割り当てを削除するか、一部を一時的な割り当てにします。 |

## セキュリティ アラート設定の構成

### "疑わしい特権ロールのアクティブ化の更新" アラート

**[アクティブ化の更新の時間枠]** 設定と **[Number of activation renewals]** (アクティブ化の更新の数) 設定を構成して、このアラートがトリガーされるタイミングを制御します。

1. ダッシュボードの **[アクティビティ]** セクションで、**[セキュリティ アラート]** を選択します。**[アクティブなセキュリティ通知]** ブレードが表示されます。
2. **[設定]** をクリックします。
3. **[アクティブ化更新時間枠]** を、スライダーを調整するか、テキスト フィールドに分単位の値を入力して、設定します。最大値は、100 です。
4. アクティブ化更新時間枠内での **[アクティブ化更新回数]** を、スライダーを調整するか、テキスト フィールドに更新回数を入力して、設定します。最大値は、100 です。
5. **[保存]** をクリックします。

### "Redundant administrators" (必要以上に管理者が指定されています) アラート
1. ダッシュボードの **[アクティビティ]** セクションで、**[セキュリティ アラート]** を選択します。**[アクティブなセキュリティ通知]** ブレードが表示されます。
2. **[設定]** をクリックします。
3. スライダーを調整するか、テキスト フィールドに日数を入力して、ロールをアクティブ化しなくても許される日数を選択します。
4. **[保存]** をクリックします。

### "Too many global administrators" (グローバル管理者が多すぎます) アラート

このアラートをトリガーする可能性のある設定は 2 つあります。
- **[グローバル管理者の最低人数]** は、許可される数よりも多くの管理者がいる場合にアラートをトリガーします。
- **[グローバル管理者の割合]** は、グローバル管理者である管理者の割合が、設定で許可されている数よりも高い場合にアラートをトリガーします。

1. ダッシュボードの **[アクティビティ]** セクションで、**[セキュリティ アラート]** を選択します。**[アクティブなセキュリティ通知]** ブレードが表示されます。
2. **[設定]** をクリックします。
3. スライダーを調整するか、テキスト フィールドに値を入力して、**[グローバル管理者の最小数]** を設定します。
4. スライダーを調整するか、テキスト フィールドに割合を入力して、**[グローバル管理者の割合]** を設定します。
5. **[保存]** をクリックします。
