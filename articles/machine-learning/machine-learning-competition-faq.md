# Microsoft Azure Machine Learning コンペティションについてよく寄せられる質問

**データ サイエンスに関する一般的な質問はどこで聞くことができますか?**

[Microsoft Azure Machine Learning フォーラム](https://social.msdn.microsoft.com/forums/azure/home?forum=MachineLearning)を利用してください。

**コンペティションに参加するにはどうすればよいですか?**

[Microsoft Azure Machine Learning](https://studio.azureml.net/?selectAccess=true&o=2%22%20\t%20%22_blank) で無料または有料のアカウントを作成します。コンペティションのチュートリアルに従い、簡単なビデオを観てください。

**参加するには、データ サイエンティストである必要はありますか?**

いいえ。コンテストに参加するには、データ サイエンティストに興味があるデータのファンや、他の専門家ではないデータ サイエンティストになることをお勧めします。補助ドキュメントは、全員が完了できるように設計されています。

**どのくらいの時間がかかりますか?**

約 10 ～ 15 分でコンペティションに参加し、データ モデルを作成できます。Machine Learning の熟練度によっては、予測実験の調整に 15 分以上かかることがあります。

**スコアを計算するにはどうすればよいですか? コンペティションの実行中と終了後では計算方法は違いますか? パブリック スコアとプライベート スコアとは何ですか?**

1.  データセット全体は、ランダムにトレーニング データ (60%) とテスト データ (残りの 40%) の階層に分けられます。ランダムな分割は、地理 + セグメント + サブグループで階層化され、トレーニング データとテスト データの両方のラベル分布が各リージョン内で一貫するようにします。  

2.  トレーニング データは、Reader モジュール構成の Starter Experiment の一部としてアップロードされ、表示されています。

3.  テスト データは、同じ階層 (パブリック テストに 60%、プライベート テスト 40%) を使用して、さらにパブリックとプライベートのテスト データに分割されました。

4.  パブリック テスト データは、初回ラウンドのスコア付けに使用されました。結果はパブリック スコアと呼ばれ、エントリを送信するときに送信履歴が取得されます。この処理はエントリを送信するごとに実行されます。このパブリック スコアは、パブリック スコアボードのランク付けに使用されます。

5.  プライベート テスト データは、コンペティションの完了後、最終ラウンドのスコア付けに使用されます。これは、プライベート スコアと呼ばれます。

6.  参加者ごとに、パブリック スコアの上位 5 個のエントリが自動的に選択され、プライベート スコアが計算されます。最終的なランク付けを入力するために、最も高いプライベート スコアのエントリが選択されます。これによって最終的な入賞者が決定されます。

**参加料金はかかりますか?**

いいえ、Microsoft Azure Machine Learning にコンペティション用の無料のワークスペースを設定できます。ただし、ゲスト アカウントはデータを保存できないので、無料の 8 時間の有効期間内でエントリを送信してください。

**コンペティションで受賞するとどうなりますか?**

Microsoft は、プライベート スコアボードの結果を確認し、お客様に連絡します。ユーザー プロファイルの電子メール アドレスが最新であることを確認してください。

**コンペティションで受賞した場合、賞金を入手するにはどうすればよいですか?**

コンペティションで受賞した場合、資格、ライセンス、リリースの宣言に署名する必要があります。このフォームには、コンペティション規則が繰り返し記載されています。受賞者は、米国の納税申告フォーム W-9 (米国の納税者ではない場合はフォーム W-8BEN) に記入する必要があります。
