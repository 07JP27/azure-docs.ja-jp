# 概要: Azure Cognitive Services の話者認識 API

話者認識機能でアプリケーションを強化する方法について説明します。話者認識 API は、最も高度な話者認証および話者識別アルゴリズムを提供するクラウド ベースの API です。話者認識は、2 つのカテゴリに分類できます。

  * 話者認証
  * 話者識別

## 話者認証

人の声には固有の特性があるため、指紋と同じように、ユーザーを一意に識別することができます。アクセス制御および認証シナリオの信号として音声を使用する技術は、新しい革新的なツールとして登場し、基本的にはセキュリティ水準を高め、お客様の認証エクスペリエンスを簡素化します。

話者認証 API は人の声や会話でユーザーを自動的に確認し、認証できます。

### 話者認証の登録

話者認証の登録はテキスト依存型です。つまり、登録フェーズと検証フェーズの両方で使用する特定のパス フレーズを話者が選択する必要があります。

登録フェーズでは、特定の語句を発話する話者の声が録音されます。その後、声の特性がいくつか抽出され、選択されたフレーズが認識されます。この抽出された特性と、選択された語句の両方が一意の声紋を形成します。

### 入力された音声と語句の検証

検証フェーズでは、入力された音声とフレーズが、登録された声紋および語句と照合され、同一人物によって発話されているかどうか、また発話された語句が正しいかどうかが検証されます。

話者認証の詳細については、[話者認識 API に関するドキュメントの検証の説明](https://dev.projectoxford.ai/docs/services/563309b6778daf02acc0a508/operations/563309b7778daf06340c9652)をご覧ください。

## 話者識別

話者識別 API では、オーディオ ファイルに登録された人の声を自動的に特定できます。このファイルには、一連の予想される話者の声が保存されています。入力された音声が、ファイル内の話者グループと照合され、一致が見つかった場合は、その話者の ID が返されます。

すべての話者が、登録プロセスでそれぞれの声をシステムに登録し、声紋を作成しておく必要があります。

### 話者識別の登録

話者識別の登録はテキスト非依存型です。つまり、オーディオ ファイルに登録する語句に制限はありません。話者の声の録音後、声の特性がいくつか抽出され、これが一意の声紋を形成します。

### 話者認識: 入力された音声との照合

認識中は、一連の予測される話者と共に、不明話者のオーディオが提供されます。入力された音声は、それが誰の声かを特定するためにすべての話者と照合され、一致が見つかった場合は、その話者の ID が返されます。

話者識別の詳細については、[話者認識 API に関するドキュメントの識別の説明](https://dev.projectoxford.ai/docs/services/563309b6778daf02acc0a508/operations/5645c068e597ed22ec38f42e)をご覧ください。
