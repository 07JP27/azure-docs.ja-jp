# 仮想ネットワーク (VNet) の作成方法

VNet を作成すると、VNet 内のサービスと VM はインターネットを経由せずに相互に安全な通信を行うことができるようになります。Azure VNet の作成は、他の VNet やオンプレミス ネットワークに接続することを目的としない場合、それほど時間のかからない簡単なプロセスです。これは、VPN デバイスの取得や構成、選択した IP アドレスと他の VNet やローカル ネットワークとの調整が不要のためです。

>[AZURE.WARNING]後で他の VNet やオンプレミス ネットワークに接続される VNet を作成する場合は、この手順を使用しないでください。安全なクロスプレミス接続またはハイブリッド接続を作成する場合は、「[仮想ネットワークのセキュリティで保護されたクロスプレミス接続について](https://msdn.microsoft.com/library/azure/dn133798.aspx)」をご覧ください。別の VNet に接続する VNet を作成する場合は、「[VNet 間の接続の構成](../vpn-gateway/virtual-networks-configure-vnet-to-vnet-connection.md)」をご覧ください。

## VNet の構成

1. **Azure 管理ポータル**にログインします。

1. 画面の左下隅で **[新規]** をクリックします。ナビゲーション ウィンドウで **[ネットワーク サービス]** をクリックし、**[仮想ネットワーク]** をクリックします。**[カスタム作成]** をクリックして、構成ウィザードを開始します。

1. **[仮想ネットワークの詳細]** ページで、以下の情報を入力します。

	![仮想ネットワークの詳細](./media/virtual-networks-create-vnet/IC736054.png)

	- **[名前]** - VNet の名前を指定します。たとえば、「*EastUSVNet*」という名前を作成しました。好きな名前を作成できます。この VNet の名前は、VM やサービスをデプロイするときに使用するため、あまり複雑な名前にしないことをお勧めします。

	- **[場所]** - ドロップダウン リストから場所 (リージョン) を選択します。場所は、リソース (VM) をこの VNet にデプロイするときに配置する物理的な場所に直接関係します。たとえば、VM を物理的に "*米国東部*" に配置する場合は、その場所のリージョンを選択します。VNet を作成した後に、その VNet に関連付けられたリージョンを変更することはできません。

1. **[DNS サーバーおよび VPN 接続]** ページでは、何も変更しません。矢印をクリックして次のページに進みます。Azure には既定で、VNet 用の基本的な名前解決が用意されています。ただし、基本的な Azure の名前解決では対応できないほど名前解決の要件が複雑である場合もあります。そのような場合は、DNS を実行する仮想マシンを後で VNet に追加することも可能です。Azure の名前解決と DNS の詳細については、「[名前解決 (DNS)](https://msdn.microsoft.com/library/azure/jj156088.aspx)」をご覧ください。

1. **[仮想ネットワーク アドレス空間]** ページでは、この VNet で使用するアドレス空間を入力します。VM 用に特定の内部 IP アドレス範囲が必要な場合や、静的 DIP を受け取る VM 用に特定のサブネットを作成する場合を除き、このページで変更を行う必要はありません。複数のサブネットを作成する場合は、このページの **[サブネットの追加]** をクリックします。

	追加情報:

	- このページの範囲には、VM をこの VNet にデプロイするときに VM が受け取る動的な内部 IP アドレス (DIP) が含まれます。これらの IP アドレスは、VNet 内での通信専用です。インターネット エンドポイントの IP アドレスではありません。

	- ここではクロスプレミス VPN 構成を使用してこのプライベート VNet をオンプレミス ネットワークに接続するわけではないので、これらの設定と既存のオンプレミス ネットワークの IP アドレスの範囲を調整する必要はありません。後でクロスプレミス構成を作成する可能性がある場合は、ルーティングの問題を避けるため、アドレス空間とローカル サイト上の既存の範囲をここで調整する必要があります。範囲を後から変更すると、アドレス範囲が重なっている場合は特に、作業が多少複雑になることがあります。

	![アドレス空間](./media/virtual-networks-create-vnet/IC716778.png)

1. **[仮想ネットワーク アドレス空間]** ページの右下にあるチェックマークをクリックすると、VNet が作成されます。VNet が作成されると、管理ポータルの**ネットワーク** ページの **[状態]** に **[作成済み]** と表示されます。

1. VNet の作成後は、VNet 内に VM と PaaS インスタンスを作成できます。VNet の選択を可能にするため、新しい VM を作成する際に **[ギャラリーから]** を必ず選択するようにしてください。既存の VM と PaaS インスタンスが既にデプロイされている場合は、それらを新しい VNet に移動する前に必要な手順があることにご注意ください。これは、必要になるネットワーク構成の設定がデプロイ中に追加されるためです。

## VNet への仮想マシンの追加

作成した VNet には、新しい VM を追加できます。重要なのは、最初に VNet を作成してから VM をデプロイすることです。VM をデプロイした後に VNet へ移動するには、再デプロイする必要があります。管理ポータルを使用して VM を作成する場合、VM を VNet にデプロイするインターフェイスは、**[新規]、[Compute]、[仮想マシン]、[ギャラリーから]** を選択したときにのみ使用可能です。VM を作成するウィザードを完了すると、**[仮想マシンの構成]** ページに **[リージョン/アフィニティ グループ/仮想ネットワーク]** が表示されます。ドロップダウン リストで、既に作成した VNet を選択します。仮想マシン作成の詳細については、[Azure 仮想マシン](../virtual-machines)に関するページを参照してください。
