# サービス要求情報に関するトラブルシューティング

Azure モバイル エンゲージメント サービス要求を開いたら、次の情報を記入してください。
 
## ID: 問題に関連する適切な ID を記入してください。
- アプリケーション ID
- キャンペーン ID
- デバイス ID
- ユーザー ID
- ユーザー名
- アプリ情報のタグ
 
## エラー： 問題に関連する適切なエラー情報を記入してください。
- 問題が発生した API や UI のセクションの名前
- 受信するすべてのエラー メッセージのテキスト
- 「[トラブルシューティング ガイド](http://go.microsoft.com/fwlink/?LinkId=524382)」に従って実行したテストの結果


## コード： 問題に関連する適切なコーディング情報を記入してください。
- アプリの SDK のバージョンとプラットフォーム \(Android SDK 2.4.1、iOS 1.16.2 などです。\)
- 実稼働アプリケーション \(開発環境アプリの APK/TGZ ファイル\) のダウンロード場所
- "AndroidManifest.xml" や Azure モバイル エンゲージメントサービスに関連するアプリからのすべてのコード スニペット \(高度なトラブルシューティング用\)
