# Azure RemoteApp とは

RemoteApp を使用すると、Azure 経由でリモートでアクセスしているプログラムを、エンド ユーザーのローカル コンピューターで実行しているかのように表示できます。これらのプログラムは、RemoteApp プログラムとして呼ばれます。ユーザーに対してリモート デスクトップ セッション ホスト (RD セッション ホスト) サーバーのデスクトップに表示されるのではなく、RemoteApp プログラムはクライアントのデスクトップに統合されます。RemoteApp プログラムはサイズ変更可能な専用のウィンドウ内で実行され、複数のモニターの間をドラッグできます。タスク バーには専用のアイコンが表示されます。ユーザーが複数の RemoteApp プログラムを同じ RD セッション ホスト サーバーで実行すると、RemoteApp プログラムは同じリモート デスクトップ サービス セッションを共有します。

RemoteApp を使用すると、次のように、多くの状況で複雑さを軽減し、管理オーバーヘッドを削減することができます。

-   支社でローカル IT サポートやネットワーク帯域幅が制限されている場合。
-   ユーザーがリモートでプログラムにアクセスする必要がある状況。
-   基幹業務 (LOB) プログラム、特にカスタム LOB プログラムのデプロイメント。
-   ユーザーにコンピューターが割り当てられていない、"ホット デスク" や "ホテリング" ワークスペースなどの環境。
-   複数のバージョンのプログラム、特に複数のバージョンをローカルでインストールすると競合を起こすようなプログラムのデプロイメント。

従来のリモート デスクトップ サービスとは異なり、Azure RemoteApp は Azure の管理ポータルで実行されます。ユーザーがポータルを使用してプログラムにアクセスするのに対して、管理者は管理ポータルを使用してすべてのプログラムとユーザーを管理します。

RemoteApp のデプロイメントには、次の 2 種類の方法があります。

-   クラウド デプロイメントでは、Azure クラウドにホストされ、プログラムのすべてのデータがクラウドに格納されます。
-   ハイブリッド デプロイメントでは、Azure クラウドにホストされますが、ユーザーはローカル ネットワークに格納されたデータにアクセスできます。

選択したデプロイメントの種類に関係なく、サービスを作成した後は、ユーザーと共有するプログラムをエンド ユーザーのフィードに公開します。これは、ユーザーが Azure ポータルからアクセスする、使用可能なプログラムの一覧です。フィードにはサブスクリプションに関連付けられたすべてのサービスからすべてのプログラムが表示されることに注意してください。
