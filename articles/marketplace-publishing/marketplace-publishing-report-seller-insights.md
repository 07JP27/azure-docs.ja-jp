# Seller Insights レポートの概要

**Seller Insights とは**

提供品で使用が発生した場合、すべての VM および使用量ベース課金制開発者サービスのパブリッシャーは、Microsoft から毎月レポートを受け取ります。

**受け取るレポート**

- **ウェルカム メール:** 新しいパブリッシャーは、Seller Insights マンスリー レポートの提供が始まることを通知するウェルカム メールを受け取ります。

- **マンスリー売上レポート:** 使用実績がある場合は、レポートとパスワードにアクセスする方法が添付された 2 番目の電子メールを受け取ります。

    - VM がある場合、または使用量ベース課金制開発者サービス SKU の使用実績ある場合は、BYOL 以外の提供品の注文、使用量、市場、顧客に関する詳細がマンスリー レポートで示されます。
    - これらのファイルは、顧客データを保護するため、パブリッシャーと Microsoft だけが知っているパスワードによってロックされています。
    - 該当月にどの提供品にも使用実績がない場合、レポートは送られません。

## Seller Insights レポートを読む方法


**SKU およびライセンス タイプ別の注文数: [Marketplace 注文] タブ**

![readingreportbyorders][2]

- スライサーを使用して各要素でレポートをフィルター処理できます。
- Azure ライセンス タイプごとに月別の注文数を示すチャートがあります。各バーはその月の合計注文数を示し、Azure ライセンス タイプ別に細分されています。
- SKU ごとに月別の注文数を示すチャートがあります。各バーはその月の全 SKU の合計注文数を示し、SKU 別に細分されています。
- Azure ライセンス タイプ別および Marketplace ライセンス タイプ別に注文数の月ごとの傾向を示すチャートがあります。
- Azure ライセンス タイプ別および Marketplace ライセンス タイプ別の注文数を示す円グラフがあります。
- Marketplace ライセンス タイプ別の月ごとの合計注文数、月ごとの合計、すべての月の総計を示す表があります。


**SKU およびライセンス タイプ別の注文数: [Marketplace 利用状況] タブ**

![readingreportbyusage][3]

- スライサーを使用して各要素でレポートをフィルター処理できます。
- 正規化された VM の利用状況情報または処理されていない利用状況データのいずれかを選択する必要があります。
- Azure ライセンス タイプごとに月別の利用状況を示すチャートがあります。各バーはその月の合計利用状況を示し、Azure ライセンス タイプ別に細分されています。
- SKU ごとに月別の利用状況を示すチャートがあります。各バーはその月の全 SKU の合計利用状況を示し、SKU 別に細分されています。
- Azure ライセンス タイプ別および Marketplace ライセンス タイプ別に利用状況の月ごとの傾向を示すチャートがあります。
- Azure ライセンス タイプ別および Marketplace ライセンス タイプ別の利用状況を示す円グラフがあります。
- Marketplace ライセンス タイプ別の月ごとの合計利用状況、月ごとの合計、すべての月の総計を示す表があります。


**[注文データ] タブおよび [利用状況データ] タブ**

これらのタブでは、レポートの生成に使われた詳細なデータが提供されます。

![orderdata][4]

![usagedata][5]



**SKU およびライセンス タイプ別の利用状況: [顧客] タブ**

![customerstab][6]

- 機密性に関する記述に注意してください
- SKU 別の顧客のリスト、プロファイル情報、取引の日付、およびプロモーションの連絡に関するオプトインが含まれます。
- レポートには、SKU 別の注文数および合計が含まれます。

## パスワードのリマインダーを要求する方法

**https://publish.windowsazure.com/ に移動し、Microsoft アカウントの資格情報でサインインします** ![passwordreminder][7]

**[パブリッシャー] タブを選択します** ![selectpublisherstab][8]


**URL でパブリッシャー ID を探します** - この ID をパスワードとして使用して、Seller Insights の Excel ファイルを開きます。通知があるまで、引き続きこの IT がパスワードとして使用されます。さらに、ブックを読むときは Windows で MS Office 2013 を使うことをお勧めします。MS Office for Mac を使うと問題があることが報告されています。

![publisherid][9]

URL でパブリッシャー ID を探します。この ID をパスワードとして使用して、Seller Insights の Excel ファイルを開きます。通知があるまで、引き続きこの IT がパスワードとして使用されます。さらに、ブックを読むときは Windows で MS Office 2013 を使うことをお勧めします。MS Office for Mac を使うと問題があることが報告されています。

## 次のステップ  
レポートおよび Insights に関して質問がある場合は、サポート チームにお問い合わせください。

1. サポート ページ (https://publish.windowsazure.com/) に移動します
2. [レポートと Insights] を選択します
3. [レポートに関する質問] を選択します
4. [要求の開始] をクリックします![sellerinsightsquestions][10]
