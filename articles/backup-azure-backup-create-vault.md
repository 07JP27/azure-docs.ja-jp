# バックアップ コンテナーを作成します。
バックアップするファイルおよびデータを Windows Server または System Center データ Protection Manager (SCDPM) を Azure にまたはを Azure に IaaS Vm をバックアップするときに、データを保存するリージョンにバックアップ コンテナーを作成する必要があります。

このチュートリアルは、バックアップの格納に使用するコンテナーの作成に使用します。

1. サインイン、 [管理ポータル](https://manage.windowsazure.com/)
2. をクリックして **新規** ]-> [ **Data Services** ]-> [ **復旧サービス** ]-> [ **バックアップ資格情報コンテナー** 選択 **簡易作成** <br/> ![コンテナーを作成します。][1]

3.  **名前** パラメーターで、バックアップ コンテナーを識別する表示名を入力します。これは、サブスクリプションごとに一意である必要があります。

4.  **領域** パラメーターで、バックアップ コンテナーのリージョンを選択します。選択は、バックアップ データが送付先となる地域を決定します。Azure にバックアップする場合、地理的に近い、場所を選択すると、ネットワーク待機時間を削減できます。

5. クリックして **資格情報コンテナーの作成** 、ワークフローを完了します。バックアップ資格情報コンテナーが作成されるまで時間がかかることがあります。状態を確認するには、ポータルの下部に、通知を監視できます。 <br/> ![資格情報コンテナーを作成します。][2]

6. バックアップ コンテナーが作成された後を示すメッセージが、コンテナーが正常に作成され、復旧のサービスとしてのリソースに表示されます **Active**. <br/> ![コンテナーの状態を作成します。][3]


## Azure のバックアップ - Storage 冗長オプション

最適な時期、ストレージ冗長性オプションを識別するためには、資格情報コンテナーの作成後に」と「すべてのマシンは、資格情報コンテナーに登録する前にです。項目が、資格情報コンテナーに登録されているし、記憶域の冗長性のオプションがロックされている変更することはできません。

ビジネス ニーズでは、Azure Backup のバックエンド ストレージの記憶域の冗長性を決定します。バックアップ記憶域のプライマリ エンドポイントとして Azure を使用しているかどうか (例: をバックアップする Azure に、Windows Server から)、地理冗長ストレージ オプションの (既定値) を選択する必要があります。これには、表示、 **構成** 、バックアップ資格情報コンテナーのオプション <br/> 。![GRS][4]

### 地理冗長ストレージ (GRS)
GRS では、データのコピーが 6 つ保持されます。GRS を使用すると、データがプライマリ リージョン内で 3 回複製され、プライマリ リージョンから数百マイル離れたセカンダリ リージョンでも 3 回複製されます。そのため、最も優れたレベルの持続性が実現されます。Azure のバックアップにより、GRS でデータを格納することで、プライマリ リージョンで障害が発生した場合は 2 つの別個の領域で、データの持続性のあります。

### ローカル冗長ストレージ (LRS)
ローカル冗長ストレージ (LRS) では、データのコピーが 3 つ保持されます。LRS は、1 つのリージョンの 1 つの施設内で 3 回複製されます。LRS は、全体の Azure の機能の障害からではなく一般的なハードウェア障害からデータを保護します。

第 3 のバックアップ ストレージ エンドポイントとして Azure を使用しているかどうか (SCDPM ローカルのバックアップを使用するなど内部設置型と、長期保存の必要があるは、Azure を使用してコピー)、ローカル冗長ストレージからの選択を検討する必要があります、 **構成** 、バックアップ資格情報コンテナーのオプションです。これにより、Azure では、データを格納する、第 3 のコピーの許容される可能性のあるデータの持続性の低いレベルを提供しつつのコストを削減します。 <br/> ![LRS][5]
