# Azure セキュリティ センターの概要

このドキュメントでは、Azure セキュリティ センターのセキュリティ監視およびポリシー管理コンポーネントの概要と、次のステップへの展開について説明します。

> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。このドキュメントでは、サンプルのデプロイを使用してサービスについて紹介します。ステップ バイ ステップ ガイドではありません。

## Azure セキュリティ センターとは
 セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

## 前提条件

セキュリティ センターを使用するには、Microsoft Azure のサブスクリプションが必要です。セキュリティ センターは、サブスクリプションがある場合に有効です。サブスクリプションがない場合は、[無料試用版](https://azure.microsoft.com/pricing/free-trial/)にサインアップできます。

 セキュリティ センターには [Azure ポータル](https://azure.microsoft.com/features/azure-portal/)からアクセスします。詳細については、[ポータルのドキュメント](https://azure.microsoft.com/documentation/services/azure-portal/)を参照してください。


## セキュリティ センターへのアクセス

セキュリティ センターにアクセスするには、ポータルで次の手順を実行します。

1. **[参照]** を選択し、**[セキュリティ センター]** オプションまでスクロールします。 ![Access Azure Security Center in the portal][1]

2. **[セキュリティ センター]** を選択します。**[セキュリティ センター]** ブレードが開きます。
3. 後で **[セキュリティ センター]** ブレードにアクセスしやすいように、右上にある **[ダッシュボードにブレードをピン留め]** オプションを選択します。 ![Pin blade to dashboard option][2]

## セキュリティ センターの使用

サブスクリプションのセキュリティ **ポリシー**を構成します。

4. **[セキュリティ センター]** ブレードの **[セキュリティ ポリシー]** タイルをクリックします。
5. **[セキュリティ ポリシー – サブスクリプションごとにポリシーを定義]** ブレードで、サブスクリプションを選択します。
6. **[セキュリティ ポリシー]** ブレードで、**[データの収集]** をオンにして、自動的にログを収集します。また、**[データの収集]** をオンにすると、サブスクリプションの既存の VM と新しい VM のすべてで、監視拡張機能がプロビジョニングされるようになります。
7. **[ストレージ アカウントの選択]** をクリックします。実行している仮想マシンがある各リージョンに対し、これらの仮想マシンから収集されたデータが格納されているストレージ アカウントを選択します。リージョンごとにストレージ アカウントを選択しないと、自動的に作成されます。収集されたデータは、セキュリティ上の理由で他の顧客データから論理的に分離されます。
8. セキュリティ ポリシーの一部として表示する **[推奨事項]** をオンにします。次に例を示します。

  - **[システムの更新]** をオンにすると、サポートされているすべての仮想マシンで不足している OS の更新プログラムがスキャンされます。
  - **[基準ルール]** をオンにすると、サポートされているすべての仮想マシンがスキャンされ、攻撃に対する仮想マシンの脆弱性を高める可能性がある OS の構成が特定されます。

![The security policy tile in Azure Security Center][3]

**推奨事項**を処理します。

9. **[セキュリティ センター]** ブレードに戻り、**[推奨事項]** タイルをクリックします。セキュリティ センターは、Azure リソースのセキュリティの状態を定期的に分析します。潜在的なセキュリティの脆弱性が識別されると、推奨事項がここに表示されます。
11.	各推奨事項をクリックすると、詳細な情報を表示したり、問題を解決するためのアクションを実行したりできます。

![Recommendations in Azure Security Center][4]

**[リソース ヘルス]** で、リソースの正常性とセキュリティの状態を表示します。

12.	**[セキュリティ センター]** ブレードに戻ります。
13.	**[セキュリティ ヘルス]** タイルには、**[仮想マシン]**、**[ネットワーク]**、**[SQL]**、**[アプリケーション]** についてのセキュリティ状態のインジケーターがあります。
14.	**[仮想マシン]** を選択して詳細な情報を表示します。
15.	**[仮想マシン]** ブレードにステータスの概要が表示され、仮想マシンのマルウェア対策プログラム、システムの更新、再起動、基準ルールの状態を確認できます。
16.	**[防止手順]** の下にある項目を選択すると、詳細情報を表示したり、必要な制御を構成するアクションを実行したりできます。
17.	特定の仮想マシンの詳細な情報を表示するにはドリルダウンします。

![The Resources health tile in Azure Security Center][5]

**セキュリティ アラート**を処理します。

19.	**[セキュリティ センター]** ブレードに戻り、**[セキュリティ アラート]** タイルをクリックします。**[セキュリティ アラート]** ブレードには、アラートの一覧が表示されます。アラートは、セキュリティ センターがセキュリティ ログとネットワークの活動を分析して生成されたものです。統合されているパートナー ソリューションからのアラートも含まれます。 ![Security alerts in Azure Security Center][6]

21.	アラートをクリックすると詳細な情報が表示されます。

  ![Security alert details in Azure Security Center][7]
