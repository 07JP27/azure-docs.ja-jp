#Azure セキュリティ センターでのセキュリティ ヘルスの監視
このドキュメントは、Azure セキュリティ センターで監視機能を使用してポリシーのコンプライアンスを監視するために役立ちます。

> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。

## Azure セキュリティ センターとは
Azure Security Center は、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

##セキュリティのヘルスの監視とは
私たちは監視について、状況に対処できるように、イベントが発生するのを監視して待機することだと考えがちです。セキュリティの監視は、組織の標準やベスト プラクティスを満たしていないシステムを識別するために、リソースを監査するプロアクティブな戦略を持つことを意味します。

##セキュリティ ヘルスの監視
サブスクリプションのリソースに対して[セキュリティ ポリシー](security-center-policies.md)を有効にすると、Azure セキュリティ センターは、潜在的な脆弱性を識別するために、リソースのセキュリティを分析します。ネットワーク設定に関する情報はすぐに利用可能になりますが、セキュリティの更新ステータスや OS の構成などの仮想マシンの構成に関する情報は、利用可能になるまで 1 時間以上かかる場合があります。**[リソースのセキュリティ ヘルス]** ブレードで、リソースのセキュリティの状態と、問題がある場合はその問題を確認することができます。これらの問題の一覧は、**[推奨事項]** ブレードでも確認できます。

推奨事項の適用方法の詳細については、「[Azure セキュリティ センターでのセキュリティに関する推奨事項の実装](security-center-recommendations.md)」を参照してください。

**[リソース ヘルス]** タイルで、リソースのセキュリティの状態を監視することができます。次に示すように、注意を必要とする高レベルと中レベルの重大度を持つ問題の数が表示されます。

![リソース ヘルス](./media/security-center-monitoring/security-center-monitoring-fig1-new.png)

有効になっているセキュリティ ポリシーが、ここで監視されているコントロールのタイプに影響します。Azure セキュリティ センターが、セキュリティ更新プログラムがない VM や[ネットワーク セキュリティ グループ](virtual-networks-nsg.md)がないサブネットなど、対処する必要がある脆弱性を識別する場合は、ここで識別します。

###仮想マシン
**[リソース ヘルス]** タイルで **[Virtual Machines]** をクリックすると、**[Virtual Machines]** ブレードが開き、次のようにオンボードおよび防止の手順に関する詳細と、Azure セキュリティ センターで監視されるすべての VM の一覧が表示されます。

![VM ごとの欠落しているシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig2-2-new.png)

このブレードが開くと、3 つのセクションが表示されます。

- 利用開始手順
- 仮想マシンの推奨事項
- 仮想マシン

各セクションでは、その問題に対処するために推奨される手順に関する詳細を表示する個別のオプションを選択できます。以下のセクションでは、これらの領域について詳しく説明します。

####利用開始手順
このセクションには、データ収集用に初期化された仮想マシンの合計容量と、データ収集の現在の状態が表示されます。各 VM のデータ収集が初期化されると、各 VM で Azure セキュリティ センターのセキュリティ ポリシーを受信する準備が完了します。このエントリをクリックすると、**[データ収集を初期化しています]** ブレードが開き、VM の名前とデータ収集の現在の状態を **[インストール状態]** 列で確認できます。

![初期化状態](./media/security-center-monitoring/security-center-monitoring-fig3-new.png)


####仮想マシンの推奨事項
このセクションには、Azure セキュリティ センターで監視されている各 VM についての推奨事項が表示されます。次のように、最初の列には推奨事項の説明、2 番目の列にはその推奨事項によって影響される VM の合計容量が表示され、3 番目の列には重要度が表示されます。

![VM 推奨事項](./media/security-center-monitoring/security-center-monitoring-fig4-2-new.png)

各推奨事項にはいくつかのアクションが示されており、アクションはクリックして実行できます。たとえば、**[不足しているシステムの更新]** をクリックすると、**[不足しているシステムの更新]** ブレードが開き、次のようにパッチが欠落している VM とそれぞれの重要度の一覧が表示されます。

![不足しているシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig5-new.png)

[不足しているシステムの更新] ブレードには、次の情報を示すテーブルが表示されます。

- **仮想マシン**: 更新プログラムが不足している仮想マシンの名前。
- **システムの更新**: 不足しているシステム更新プログラムの件数。
- **最終スキャン時刻**: Azure セキュリティ センターで VM の更新が最後にスキャンされた時刻。
- **状態**: 推奨事の現在の状態 
	- **オープン**: 未対応
	- **処理中**: 推奨事項は現在、これらのリソースに適用されており、ユーザーのアクションは不要
	- **解決済み**: 推奨事項が既に完了済み (状態が解決済みになると、行の色が灰色表示になる)
- **重要度**: 特定の推奨事項の重要度の説明 
	- **高**: 重要なリソース (アプリケーション、VM、ネットワーク セキュリティ グループ) に脆弱性が存在しており、注意が必要
	- **中**: プロセスを完了または脆弱性を排除するために重大ではないまたは追加の手順が必要
	- **低**: 対処すべきだが、直ちにアクションが必要ではない脆弱性(既定では、重要度の低い推奨事項は表示されないが、フィルターで [低] の推奨事項をオンにすれば、表示することが可能)

推奨される更新プログラムの詳細を表示するには、VM の名前をクリックします。次のように VM の新しいブレードが開き、不足している更新プログラムの一覧が表示されます。

![VM ごとの不足しているシステムの更新](./media/security-center-monitoring/security-center-monitoring-fig6-new.png)

> [AZURE.NOTE] ここのセキュリティの推奨事項は、[推奨事項] ブレードに表示されているものと同じです。解決の推奨事項の詳細については、「[Azure セキュリティ センターでのセキュリティに関する推奨事項の実装](security-center-recommendations.md)」を参照してください。これは VM だけでなく、[リソース ヘルス] タイルで利用できるすべてのリソースに適用できます。

####仮想マシン
[仮想マシン] セクションには、すべての VM と推奨事項の概要が表示されます。各推奨事項の列は、次のように推奨事項の 1 つのセットを表します。

![VM](./media/security-center-monitoring/security-center-monitoring-fig7-new.png)

各推奨事項の下に表示されるアイコンで、どの VM の、どのような種類の推奨事項に注意する必要があるかをすばやく特定できます。

前に示した例では、1 つの VM にマルウェア対策に関する重要な推奨事項があります。VM に関する詳細情報を表示するには、VM をクリックします。すると、次のように、この VM を表す新しいブレードが開きます。

![VM のセキュリティの詳細](./media/security-center-monitoring/security-center-monitoring-fig8-new.png)

このブレードには VM のセキュリティに関する詳細情報が表示されます。このブレードの下部には、推奨されるアクションと、各推奨事項の重要度が表示されます。

###ネットワーク
ネットワークの防止ステータスは、Azure セキュリティ センターによって監視されている仮想ネットワークを示します。[リソース ヘルス] タイルで **[ネットワーク]** をクリックすると、**[ネットワーク]** ブレードが開き、次のような詳細が表示されます。

![ネットワーク](./media/security-center-monitoring/security-center-monitoring-fig9-new.png)

このブレードを開くと、[ネットワークの推奨事項] と [ネットワーク] という 2 つのセクションが表示されます。
 
各セクションでは、その問題に対処するために推奨される手順に関する詳細を表示する個別のオプションを選択できます。次に、これらのセクションについて詳しく説明します。

####ネットワークの推奨事項

仮想マシン リソースのヘルス情報と同様に、このブレードでは上部に問題をまとめた一覧と下部に監視対象のネットワークの一覧が表示されます。

ネットワーク状態の内訳のセクションでは、[エンドポイントの ACL](virtual-machines-set-up-endpoints.md) が有効でない、[ネットワーク セキュリティ グループ](virtual-networks-nsg.md)が有効でない、正常なサブネット、**NSG でのアクセスが制限されていない**などの防止手順が一覧表示されます。これらの推奨事項のいずれかをクリックすると、次の例に示すような新しいブレードが開き、推奨事項に関する詳細が表示されます。

![エンドポイント](./media/security-center-monitoring/security-center-monitoring-fig10-new.png)

この例では、**[外部のパブリック エンドポイントを介したアクセスを制限する]** ブレードに、このアラートに属するネットワーク セキュリティ グループ (NSG)、この NSG が関連付けられているサブネットとネットワーク、この推奨事項の現在の状態と重要度が一覧表示されます。ネットワーク セキュリティ グループをクリックすると、次のように別のブレードが開きます。

![エンドポイントの制限](./media/security-center-monitoring/security-center-monitoring-fig11-new.png)

このブレードには、ネットワーク セキュリティ グループの情報、場所、現在有効になっている受信の規則の一覧が表示されます。このブレードの下部には、このネットワーク セキュリティ グループに関連付けられている VM が表示されます。現在開かれている望ましくないポートをブロックする受信の規則を有効にする場合、または現在の受信の規則のソースを変更する場合は、ブレードの上部にある **[受信の規則の編集]** ボタンをクリックします。

####ネットワーク
**[ネットワーク]** セクションには、VM に関連付けられているリソース グループ、サブネット、ネットワーク インターフェイスの階層が次のように表示されます。

![ネットワーク ツリー](./media/security-center-monitoring/security-center-monitoring-fig121-new.png)

このセクションでは、[リソース マネージャー ベースの VM と従来の VM](resource-manager-deployment-model.md) が別々に表示されます。そのため、(ASM または従来の) Azure Service Management または Azure Resource Management (ARM) のネットワーク機能を仮想マシンで使用できるかどうかをすばやく確認できます。この場所からネットワーク インターフェイス カードのプロパティにアクセスする場合は、サブネットを展開し、VM 名をクリックする必要があります。このアクションをリソース マネージャー ベースの VM に対して実行する場合、次のような新しいブレードが表示されます。

![ネットワーク ツリー](./media/security-center-monitoring/security-center-monitoring-fig13-new.png)

このブレードには、ネットワーク インターフェイス カードとそのカードに関する現在の推奨事項の概要が表示されます。選択した VM が従来の VM である場合は、次のようにさまざまなオプションを示す新しいブレードが表示されます。

![ACL](./media/security-center-monitoring/security-center-monitoring-fig14-new.png)

このブレードでは、パブリック ポートやプライベート ポートに変更を加えることや、この VM の ACL を作成することができます。

###SQL
**[リソース ヘルス]** タイル内の **[SQL]** をクリックすると、[SQL] ブレードが開き、監査が有効でない、透過的なデータ暗号化が有効でない、データベースの全般のヘルス状態などの問題に関する推奨事項が示されます。

![SQL リソース ヘルス](./media/security-center-monitoring/security-center-monitoring-fig15-new.png)

このいずれかの推奨事項をクリックして詳細を表示し、問題を解決するためのさらなるアクションを実行できます。次の例では、**[データベースの監査が有効でない]** の推奨事項を展開しています。

![SQL リソース ヘルス](./media/security-center-monitoring/security-center-monitoring-fig16-new.png)

**[SQL データベースの監査を有効にする]** ブレードには、SQL データベースと SQL データベースが配置されているサーバーが一覧表示されます。また、この設定がサーバーから継承されたか、このデータベース固有かが示され、この問題の現在の状態と重要度も示されます。この推奨事項に対処するためにデータベースをクリックすると、次に示すような **[監査と脅威の検出]** ブレードが開きます。

![SQL リソース ヘルス](./media/security-center-monitoring/security-center-monitoring-fig17-new.png)

この機能を有効にするには、**[監査]** オプションの **[オン]** を選択し、**[保存]** をクリックします。

###アプリケーション
Azure ワークロードに、[リソース マネージャーの VM](resource-manager-deployment-model.md) 内に配置され、公開された Web ポート (TCP ポート 80 および 443) があるアプリケーションがある場合、Azure セキュリティ センターはこれらを監視して、潜在的なセキュリティ問題を特定し、修復手順を推奨します。**[アプリケーション]** タイル内をクリックすると、**[アプリケーション]** ブレードが開き、防止手順のセクションに一連の推奨事項が示され、さらに次のように、ホスト/仮想 IP ごとのアプリケーションの内訳も示されます。

![アプリケーションのセキュリティのヘルス](./media/security-center-monitoring/security-center-monitoring-fig18-new.png)

前のリソースの他の推奨事項と同様に、推奨事項をクリックすると、問題と修復方法の詳細を確認できます。次の図に示す例は、**セキュリティ保護されていない Web アプリケーション**として識別されたアプリケーションです。安全でないと見なされたアプリケーションを選択すると、使用可能なオプションが示された別のブレードが開きます。

![アプリケーション](./media/security-center-monitoring/security-center-monitoring-fig19-new.png)

**[セキュリティ保護されていない Web アプリケーション]** ブレードには、安全でないと見なされたアプリケーションを使用しているすべての VM が一覧表示されます。一覧には、VM 名、問題の現在の状態、重要度が表示されます。この Web アプリケーションをクリックすると、**[Web アプリケーション ファイアウォールを追加する]** ブレードが開き、次に示すように、サード パーティの WAF (Web アプリケーション ファイアウォール) をインストールするためのオプションが表示されます。

![WAF の追加](./media/security-center-monitoring/security-center-monitoring-fig20-new.png)
