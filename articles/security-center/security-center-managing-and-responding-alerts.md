# Azure セキュリティ センターでのセキュリティの警告の管理と対応
このドキュメントは、Azure セキュリティ センターの機能を使用してセキュリティの警告の管理と対応することに役立ちます。

> [AZURE.NOTE] このドキュメントの情報は、Azure セキュリティ センターのプレビュー リリースに適用されます。

## Azure セキュリティ センターとは
 セキュリティ センターは、Azure リソースのセキュリティを高度に視覚化し、制御することで脅威を回避、検出し、それに対応することに役立ちます。これにより、サブスクリプション全体に統合セキュリティの監視とポリシーの管理を提供し、気付かない可能性がある脅威を検出し、セキュリティ ソリューションの広範なエコシステムと連動します。

## セキュリティの警告とは何か
 セキュリティ センターは、真の脅威を検出し、偽陽性を減らすため、Azure のリソース、ネットワーク、およびマルウェア対策プログラムとファイアウォールのようなパートナー ソリューションから、自動的にログ データを収集、分析、統合します。警告の優先順位の一覧が、セキュリティの警告として表示されます。

現在の警告は、**[セキュリティの警告]** タイルで確認できます。各警告の詳細については以下の手順に従います。

1. セキュリティ センター ダッシュ ボードに **[セキュリティの警告]** タイルが表示されます。

    ![Azure セキュリティ センターの [セキュリティの警告] タイル](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig1.png)

2.  セキュリティの警告の発生アイコンをクリックします。以下のように、警告の詳細が表示された **[セキュリティの警告]** ブレードが開きます。

    ![Azure セキュリティ センターの [セキュリティの警告] ブレード](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig2.png)

このブレードの下部に、各警告の詳細が表示されます。警告を並べ替えるには、並べ替えに使用する列をクリックします。各列の定義を以下に示します。

- **警告**: 警告の短い説明。
- **カウント**: 特定の日に検出された、特定の種類の全アラートの一覧。
- **検出元**: 警告をトリガーしたサービス。
- **日付**: イベントが発生した日付。
- **状態**: その警告の現在の状態。状態には次の 3 種類があります。
    - **アクティブ**: セキュリティの警告が検出されました。
    - **破棄済み**: セキュリティの警告はユーザーによって破棄されました。通常、この状態は、調査済みであり、軽減されたか、実際の攻撃とは見なされなかった警告に使用されます。

- **重要度**: 重大度。高、中、低で示します。

### セキュリティの警告への対応
組織では攻撃の可能性はさまざまな行動で示されます。たとえば、正規のネットワーク キャプチャを実行しているネットワーク管理者は、何らかの攻撃を開始する人と同じように見える可能性があります。他にも、誤って構成されたシステムに起因し、侵入検出システムで多くの偽陽性が発生し、これにより本物のインシデントを見つけることがより困難になることもあります。セキュリティ センターを使用してセキュリティの警告を確認すると、警告の重要度に基づいて処置を講じることができます。

そのためには、対応する警告を選択します。下の図のように、より多くの詳細が含まれた新しいブレードが右側に開きます。

![Azure セキュリティ センターのセキュリティの警告への対応](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig3.png)

この場合、トリガーされた警告が、リモート デスクトップ プロトコル (RDP) の疑わしいアクティビティを参照します。1 列目には、攻撃を受けたリソースが表示されます。2 列目には、攻撃が検出された時刻が表示されます。3 列目には、警告の状態が表示されます。4 列目には、攻撃の重大度が表示されます。この情報を確認したら、攻撃を受けたリソースをクリックします。新しいブレードが開き、以下の例のように、次の対応に関する詳細な提案が表示されます。

![Azure セキュリティ センターのセキュリティの警告に対処する方法の推奨事項](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig4.png)

> [AZURE.NOTE] セキュリティ センターから提案される修復方法は、セキュリティの警告によって変わります。場合によっては、他の Azure 機能を使用して推奨される修復方法を実行する必要があります。たとえば、このような攻撃の修復には、[ネットワーク ACL](virtual-networks-acl.md) または [ネットワーク セキュリティ グループ](virtual-networks-nsg.md) ルールを使用してこの攻撃を生成している IP アドレスをブラックリストに指定します。

### セキュリティ アラートの管理
日付、状態と重要度に基づいて警告をフィルター処理することができます。**[セキュリティの警告]** ブレードで、**[フィルター]** をクリックし、次のように必要なオプションを有効にします。

![Azure セキュリティ センターの警告のフィルター](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig5.png)

警告のフィルタリングは、ダッシュボードに表示する範囲を限定する必要がある場合に便利です。たとえば、システム内の潜在的な違反を調査するために、過去 24 時間以内に発生したセキュリティの警告を確認することができます。

ほとんどの場合、セキュリティの警告に表示されている推奨方法を適用することをお勧めします。ただし、環境に合わない誤検出の場合や、特定のリソースでは期待される動作の場合など、状況によっては警告を無視する必要があります。いずれの場合も、特定の理由から **[無視]** オプションを使用して推奨事項を非表示にすることができます。

タスクを無視するには、タスクを右クリックすると、次のように **[無視]** オプションが表示されます。

![Azure セキュリティ センターの [無視] オプション](./media/security-center-managing-and-responding-alerts/security-center-managing-and-responding-alerts-fig6.png)

警告のコレクションでは、それぞれの警告に非常に具体的な説明と修復方法があります。セキュリティ センターの警告は、攻撃シナリオに基づいています。次の攻撃シナリオに関する警告は、Microsoft エンジンによってトリガーされます。

- **ネットワーク データに対するブルート フォース攻撃の検出**: ネットワーク トラフィック データから学習する Machine Learning のモデルに基づいて検出されます。
- **エンドポイント データに対するブルート フォース攻撃の検出**: コンピューターのログに対するセキュリティ センターのクエリに基づいて検出されます。これにより、失敗した試行と成功した試行を区別できるようになります。
- **悪意のある IP アドレスと通信している VM**: セキュリティ センターが、ボットによりセキュリティ侵害され、コマンド アンド コントロール (C & C) サーバーと通信しているマシン (またはその反対) の発見に基づいて検出されます。
