# <a name="power-bi-apps-prerequisites"></a>Power BI アプリの前提条件

この記事では、Microsoft AppSource Marketplace に Power BI アプリのオファーを公開する前に満たす必要がある技術要件とビジネス要件の一覧を示します。

## <a name="technical-requirements"></a>技術的な要件

このオファー タイプに必要な主な技術資産は [Power BI アプリケーション](https://go.microsoft.com/fwlink/?linkid=2028636)です。 詳細については、[Power BI の技術資産の作成](./cpp-create-technical-assets.md)に関するページを参照してください。

Power BI でサービス アプリケーションを作成し、テストした後、Power BI が生成するアプリケーション インストール URL を保存する必要があります。 [新しい Power BI アプリのオファーを作成する場合](./cpp-create-offer.md)、これが必要になります。

## <a name="business-requirements"></a>ビジネス要件

ビジネス要件には、手続き、契約、および法律に関する義務が含まれます。

-   登録済みのクラウド マーケットプレース パブリッシャーである必要があります。 まだ登録していない場合は、「[クラウド マーケットプレース パブリッシャーになる](../../become-publisher.md)」の手順に従ってください。

    > [!NOTE] 
    > Microsoft デベロッパー センター登録アカウントと同じアカウントを使用して、[Cloud パートナー ポータル](https://cloudpartner.azure.com)にサインオンする必要があります。 複数の AppSource オファリングに対して 1 つの Microsoft アカウントのみ使用する必要があります。 個々のサービスおよびオファーに対して固有のアカウントを作成してはなりません。
 
-   商取引上合理的な方法で、顧客に技術サポートを提供していただきます。
-   AppSource にオファーが掲載されるための条件を満たすコンテンツを提供する必要があります。 詳細については、[AppSource にアプリを掲載する方法](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)に関するページを参照してください。
-   [Microsoft プライバシーに関する声明](https://privacy.microsoft.com/privacystatement)に従う必要があります。


## <a name="next-steps"></a>次の手順

すべての要件を満たした後、クラウド パートナー ポータルで [Power BI のオファーを作成](./cpp-create-offer.md)できます。
