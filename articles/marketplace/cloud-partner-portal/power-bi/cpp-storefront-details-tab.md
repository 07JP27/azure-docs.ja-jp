# <a name="power-bi-apps-storefront-details-tab"></a>Power BI アプリの [ネットショップの詳細] タブ

見込み客にマーケティング、販売、および法律に関する情報を提供するには、**[新しいオファー]** ページの **[ネットショップの詳細]** タブを使用します。 このタブでは、マーケットプレースから生成されるリードの管理方法も指定します。 この長いフォームは、次の 6 つのセクションに分かれています:**[Offer Details]\(オファーの詳細\)**、**[Listing Details]\(一覧の詳細\)**、**[Marketing Artifacts]\(マーケティングの成果物\)**、**[Legal]\(法的情報\)**、**[Customer Support]\(カスタマー サポート\)**、**[Lead Management]\(リード管理\)**。  フィールド ラベルに付いているアスタリスク (*) は、そのフィールドが必須であることを示します。


## <a name="offer-details-section"></a>[オファーの詳細] セクション

このセクションでは、アプリ ソースのオファーについての一般的な情報を入力します。

![[ネットショップの詳細] タブの [オファーの詳細] セクション](./media/offer-details-section.png)

次の表では、これらのフィールドの名前と目的について説明しています。

|   フィールド               |   説明                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **[オファーの概要]**     | アプリの簡単な目的です。 最大長は 100 文字です。                             |
| **[オファーの説明]** | アプリの説明です。 最大長は 3,000 文字で、単純な HTML 書式がサポートされています。 |
|                       |                                                                                         |


## <a name="listing-details-section"></a>[一覧の詳細] セクション

この 2 番目のセクションでは、アプリの追加のコンテキストを指定します。一般に使用される業種、最も適しているカテゴリ、交換可能な製品、関連する検索語句などです。

![[ネットショップの詳細] タブの [一覧の詳細] セクション](./media/listing-details-section.png)

 次の表では、これらのフィールドの名前と目的について説明しています。
 
|   フィールド                                  |   説明                                                        |
| --------------                           | ---------------------                                                |
| **業界**                           | アプリに最も適している業種を選択します。 アプリが複数の業種に関連している場合、これは空白のままにすることができます。      |
| **Categories (カテゴリ)**                           | アプリに関連するカテゴリを選択します。 最大 3 個を選択します。     |
| **Help link for your app (アプリのヘルプ リンク)**               | アプリのオンライン ヘルプのページへの URL です           |
| **Products your app works with (Max 3) (アプリが連携する製品 (最大 3 個))** | アプリが連携する特定の製品を列挙します。 最大 3 製品を列挙できます。 製品を列挙するには、[+] マーク ([新規] の隣) をクリックすると新しい空のテキスト フィールドが作成されるので、そこにアプリが対応する製品の名前を入力します。      |
| **Search keywords (Max 3) (検索キーワード (最大 3 個))**              | AppSource により顧客はキーワードを基づいた検索を行うことができます。 アプリケーションが顧客に表示される一連のキーワードを入力できます。 たとえば、アプリケーションが "My Emailing app" の場合、Emails、Mailing、Mail app をキーワードにすることができます。 AppSource の検索ボックスでユーザーがアプリを検索するために使用する可能性のある単語を選択します。 |
|  |  |


## <a name="marketing-artifacts-section"></a>[マーケティングの成果物] セクション

この 3 番目のセクションでは、ブランドとマーケティングの資料をアップロードできます。  次の 4 つのサブセクションに分かれています:**[Logos]\(ロゴ\)**、**[Videos]\(ビデオ\)**、**[Documents]\(ドキュメント\)**、**[Screenshots]\(スクリーンショット\)**。 必須のマーケティング成果物はロゴとスクリーンショットですが、顧客へのアピールを最大にするために、すべて設定することをお勧めします。

![[ネットショップの詳細] タブの [マーケティングの成果物] セクション](./media/marketing-artifacts-section.png)

 
|    フィールド                             |    説明                                                    |
|   -----------                        |    -------------                                                  |
| *ロゴ*                              |                                                                   |
| **Offer logo (png format, 48x48) (オファーのロゴ (png 形式、48 x 48))**   | AppSource で検索が完了したときに、アプリまたはアプリの結果の概要に表示されます。 解像度が 48 \* 48 ピクセルの png 形式のみがサポートされます。  |
| **Offer logo (png format, 216x216) (オファーのロゴ (png 形式、216 x 216))** | AppSource でアプリの詳細ページに表示されます。  解像度が 216 \* 216 ピクセルの png 形式のみがサポートされます。  |
| *ビデオ*                             |                                                                   |
| **Name**                             | アプリの名前またはタイトルです                                          |
| **URL**                              | YouTube または Vimeo でホストされているビデオの URL です                              |
| **サムネイル**                        | アプリのサムネイル画像です。  解像度が 1280 \* 720 ピクセルの png 形式のみがサポートされます。   |
| *ドキュメント*                          | 省略可能ですが、最大で 3 つのドキュメントです。 ここでアップロードしたドキュメントは、AppSource の [Learn more]\(詳細\) に表示されます。  |
| **Name**                             | サポート ドキュメントの名前またはタイトルです                              |
| **ファイル**                             | アップロードするドキュメントは、pdf 形式である必要があります                             |
| *スクリーンショット*                        | 省略可能。ただし、最大 5 つのスクリーンショットです。                        |
| **Name**                             | スクリーンショットの名前またはタイトルです                                       |
| **Image**                            | アップロードする画面キャプチャ画像です。解像度が 1280 \* 720 ピクセルの png 形式でなければなりません  | 
|   |   |


### <a name="logo-guidelines"></a>ロゴのガイドライン

[Cloud パートナー ポータル](https://cloudpartner.azure.com)にアップロードされるすべてのロゴは、以下のガイドラインに従っている必要があります。

- ロゴではグラデーションの背景を使用しないでください。
- ロゴには、会社やブランド名を含むテキストを使用しないでください。 ロゴのルック アンド フィールは "フラット" にする必要があり、グラデーションは避ける必要があります。
- ロゴを拡大しないでください。


## <a name="legal-section"></a>[法的情報] セクション

この 4 番目のセクションでは、各オファーに必須の次の 2 つの法的ドキュメントを指定することができます:プライバシー ポリシーと利用規約。

![[ネットショップの詳細] タブの [法的情報] セクション](./media/legal-section.png)

|   フィールド                |   説明                           |
|------------------------|--------------------------------------   |
| **プライバシー ポリシーの URL** | 投稿済みのプライバシー ポリシーの URL       |
| **使用条件**       | プレーンテキストまたは単純な HTML のポリシーです     |
|  |  |


## <a name="customer-support-section"></a>[カスタマー サポート] セクション

オンライン カスタマー サポート ページの**サポート URL**を指定します。  このオンライン サポート ページでは、電話、電子メール、ライブ チャットなど、複数の連絡先オプションを顧客に提供することをお勧めします。 


## <a name="lead-management-section"></a>[リード管理] セクション

最後のセクションでは、AppSource オファーから生成される顧客リードを収集することができます。 このリード情報に対して、(ドロップダウン リストで) 以下のストレージ オプションが用意されています。

|    フィールド               |   リードの宛先                               |
|------------------------|--------------------------------------            |
|  **なし**              | リードを収集しません (既定値)。  |
| **Azure Blob (deprecated) (Azure Blob (非推奨))** | [Azure Blob](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) のコンテナー名と接続文字列を指定します。  この選択肢は非推奨です。代わりに、**[Azure Table]\(Azure テーブル\)** を使用してください。  |
| **Azure テーブル**        | [Azure テーブル](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview)の接続文字列を指定します  |
| **Dynamics CRM Online** | [Microsoft Dynamics 365 Online](https://dynamics.microsoft.com/) インスタンスの URL と認証資格情報を指定します |
| **HTTPS エンドポイント**     | JSON ペイロードとしての HTTPS エンドポイントを指定します   |
| **Marketo**            | [Marketo](https://www.marketo.com/) インスタンスのサーバー ID、munchkin ID、フォーム ID を指定します   |
| **Salesforce**         | [Salesforce](https://www.salesforce.com/) データベースのオブジェクト識別子を指定します |
|  |  |

オファーを発行した後は、リードの接続が検証され、指定した格納先にテスト リードが自動的に送信されます。 リード情報は継続的に管理する必要があり、現在の顧客管理アーキテクチャに合わせて、これらの設定をすぐに更新する必要があります。
