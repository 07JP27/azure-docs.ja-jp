# <a name="what-is-the-cloud-partner-portal"></a>クラウド パートナー ポータルとは
クラウド パートナー ポータルにより、独立系ソフトウェア ベンダー (ISV) が Microsoft クラウド上にアプリケーションを発行し、提供できます。 [AppSource](https://appsource.microsoft.com/) 上のビジネス ユーザーや [Azure Marketplace](https://azuremarketplace.microsoft.com/) 上の開発者および IT プロフェッショナルをターゲットにしたアプリを構築する ISV にとって、クラウド パートナー ポータルは最適な場所です。

Azure の発行元にとって、Azure Marketplace は、クラウド ベースのアプリケーションやモバイル ソリューションを短期間で開発しようとしている他の開発者、ISV、および IT プロフェッショナルに革新的なアプリケーションやサービスを配布し販売するしくみです。 さらに AppSource を使用して、お客様は業界の新しい基幹業務アプリを見つけたり、既に使用中のソリューションをもっと活用できるようにするアプリを見つけることができます。 アプリの発行により、ビジネスを遂行するためのより優れた方法を求めている新規顧客を獲得できます。

新規 ISV の方で、まだ実行されていない場合は、さっそく以下を実行してください。
- 開発者および IT プロフェッショナル向けにプランを提供する場合は、[Azure Marketplace にアプリを送信する](https://azuremarketplace.microsoft.com/sell)。 
- ビジネス ユーザー向けにプランを提供する場合は、[AppSource にアプリを送信する](https://appsource.microsoft.com/partners)。 

送信が承認され、アプリの配布準備が完了すると、Microsoft から招待メールが送信され、[クラウド パートナー ポータル](https://cloudpartner.azure.com/)にアクセスして作業を開始できるようになります。

> [!NOTE]
> クラウド パートナー ポータルは、現在**プレビュー**段階にあり、すべての製品の種類がサポートされているわけではありません。 サポートされている製品の完全な一覧については、[サポートされている製品の種類](./Cloud-partner-portal-products-that-can-get-published-via-portal.md)を参照してください。

### <a name="how-the-publishing-process-works"></a>発行プロセスのしくみ
クラウド パートナー ポータルでは、できるだけ簡単にビルド済みアプリケーションを取得して、Marketplace 上に表示できるようになっています。 発行プロセスに移行する前に完全なアプリをビルドすることを強くお勧めします。これにより、必要な操作が簡素化され、アプリがすばやく利用可能になります。

![発行のフロー図](./media/cloud-partner-portal-what-is-the-cloud-partner-portal/publishing-flow-diagram.png)


1.    新しいプランを**ビルド**するには、製品の種類を選択し、技術およびマーケティング情報を追加してプランを作成します。
2.    次にプランを**発行**します。これにより、プランが 送信されて、Marketplace の認定を受けて承認されます。 運用を開始する前に、プレビューで、Marketplace 内とまったく同じルック アンド フィールを確認できます。
3.    プランを送信して Marketplace で運用を開始しても、プロセスはまだ始まったばかりです。 プランによって何が行えるかを潜在顧客に伝え、洞察を共有することにより、ビジネスの**拡大**を可能にします。これにより、マイクロソフトとのビジネスを成功裏に構築できます。


これで発行プロセスについて理解されたので、次に[クラウド パートナー ポータルの概要](./cloud-partner-portal-getting-started-with-the-cloud-partner-portal.md)に進みます。
