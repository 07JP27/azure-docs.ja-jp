# <a name="azure-saas-application-offer"></a>Azure SaaS アプリケーション プラン

<table> <tr> <td>このセクションは、Microsoft <a href="https://azuremarketplace.microsoft.com/marketplace/">Azure Marketplace</a> または <a href="https://appsource.microsoft.com">Microsoft AppSource</a> にサービスとしてのソフトウェア (SaaS) アプリケーションを公開する方法を説明します。 ご利用の Azure サブスクリプションでソリューションがデプロイされるとき、およびデザインしたインターフェイスを通じてお客様がログオンし、アプリケーションのテストを管理するときに、SaaS アプリケーションを使います。 <a href="https://azure.microsoft.com/services/active-directory">Azure Active Directory</a> (AAD) は、既存の評価環境を活用するために使用されます。 </td> <td><img src="./media/cloud-partner-portal-publish-web-app/saasapp-icon1.png"  alt="Azure SaaS application icon" /></td> </tr> </table>


## <a name="publishing-benefits"></a>公開の利点

次に、Microsoft のマーケットプレースにアプリケーションを公開する利点をいくつか挙げます。

- Office 365 および Dynamics 365 にわたる 1 億の Azure Active Directory ユーザーにリーチできます。
- 販売チームの拡張: 世界中のビジネス ユーザーにリーチする、エンドユーザーを取り込む販売チャネルを獲得する、潜在顧客を生み出すことに役立つ、業種を超えた新しい顧客との会話のきっかけを作ることができます。

- アクションにつながる分析情報を得る: AppSource でアプリがどのように実行され、どのくらいうまく機能しているか、および営業の手順をさらに改善する方法についての分析情報を共有します。


## <a name="publishing-process"></a>公開プロセス

このセクションでは、以降の手順に従って、アプリを作成し、Microsoft Azure Marketplace または Microsoft AppSource に公開する方法を説明します。

1. パートナーの登録など、前提条件を満たしていることを確認します:[デベロッパー センター登録](https://dev.windows.com/en-us/registration?accountProgram=Azure)*または* [Microsoft Partner Network 登録](https://partner.microsoft.com/)。
2.  マーケットプレースのオファーとプランを定義します。
3.  マーケティング コンテンツと価格を指定します。
4.  プランを [Cloud パートナー ポータル](https://cloudpartner.azure.com)でテストおよび認定します。
5.  プランを公開します。
