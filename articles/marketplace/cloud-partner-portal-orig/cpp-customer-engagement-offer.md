# <a name="dynamics-365-for-customer-engagement-offer"></a>Dynamics 365 for Customer Engagement プラン

<table> <tr> <td>このセクションでは、Dynamics 365 for Customer Engagement ソリューションを Microsoft <a href="https://appsource.microsoft.com">AppSource Marketplace</a> に公開する方法について説明します。 Dynamics 365 for Customer Engagement のすべてのアプリ (販売、サービス、プロジェクト サービス、およびフィールド サービス) が当社の認定プロセスを通過し、試用版エクスペリエンスをサポートしている必要があります。 認定プロセスでは、ソリューションのベスト プラクティスについて調べます。 試用版エクスペリエンスでは、ユーザーがソリューションをライブの Dynamics 365 環境にデプロイできます。 </td> <td><img src="./media/dynce-icon1.png"  alt="Dynamics 365 Customer Engagement icon" /></td> </tr> </table>

次のビデオでは、 [Microsoft AppSource (Build 2018) で説得力のあるエクスペリエンスを提供し](https://www.youtube.com/watch?v=WWBFuNHC0J4)、AppSource マーケットプ レースの概要とこのエコシステムのメリットについて説明します。 次に、Dynamics 365 Customer Engagement カスタム アプリケーションの例と、体験版を提供することのメリット、そして AppSource に公開する方法について説明します。

## <a name="publishing-process"></a>公開プロセス

ここからは、Dynamics 365 for Customer Engagement ソリューションを作成および公開するプロセスを手順ごとに説明します。

1. Microsoft Partner Network (MPN) で[登録します](https://partners.microsoft.com/PartnerProgram/simplifiedenrollment.aspx) (まだ登録していない場合)。
2. [Dynamics 365 ソリューションを作成します](./dynamics-365-solution-preparation.md)。
3. [AppSource パッケージを作成して](./appsource-package-preparation.md)マーケティング コンテンツを関連付けます。
4.  AppSource パッケージを Azure ストレージにアップロードして[ SAS キーを生成します](./store-your-appsource-package-to-azure-storage-and-generate-sas-key.md)。
5. [関連するプランを作成します](./setup-your-application-in-the-publishing-portal.md)。
6. [プランを公開して](./request-to-publish-the-app.md)ライブにします。
