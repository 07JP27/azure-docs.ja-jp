# <a name="build-your-landing-page"></a>ランディング ページの作成  
会社の Web サイトのランディング ページは、需要を開拓する活動を経てマーケットプレイスのリストに到達するための入り口です。  

最初に、目標を明らかにします。 ソリューションでは、対象顧客とその顧客が実行する操作を選択します。  ソリューションには、次の操作が含まれます。  
*   ソリューションを体験する  
*   今すぐソリューションを入手する  
*   その他にも用途はあります。  

複数のマーケティング戦略にわたるランディング ページを参照できます。 また、マーケティング戦略を活用できます。  
*   イベント  
*   Web キャスト  
*   ソーシャル メディア  
*   ホワイト ペーパー  
*   技術面のトレーニング セッション  
*   プレス リリース  

メッセージと行動喚起に高い一貫性を持たせることで、顧客がソリューションを発見しやすくなります。  

キャンペーンの計画が準備できている場合は、ランディング ページを作成します。 ランディング ページの有効性を最大限に高めるために、次の表を確認してください。  

| ベスト プラクティス | 避けるべきこと |  
|:--- |:--- |  
| ***そのソリューションで解決される顧客の問題と、Azure がそれを実現する方法が適合している*** |  ソリューションと Azure が連携するしくみがうまく適合しない。 |  
| ***短くて覚えやすい URL を作成する*** | 長い URL を使用する。 長い URL は覚えにくく、探すのが難しい。 |  
| ***関連するビジュアル コンテンツを追加する (お客様の声のビデオやソリューションのアーキテクチャなど)*** | 使用するテキストが多すぎると、顧客はコンテンツに目を留めず、探索しなくなる。 |  
| ***訪問者を Marketplace のカタログ ページに誘導する明確な行動喚起を作成する*** | ページに用意されたリンクやアクションが多すぎる。 |  
| ***ヘッダーまたは明確に区別されたセクションに行動喚起を配置する*** |  マーケットプレイスのリストへのリンクをテキストの段落に埋め込む。 |  
| ***最適なキーワードを調査し、ページを検索に最適化する*** | 製品名を使えば高い検索順位になるものと思い込む。 |  
| ***広告キャンペーンのキーワードを使用する*** |  Web プロパティで多くの異なるキーワードを使うと、広告への投資が薄められてしまう。 |  
| ***関連する製品名とキーワードを配置する`above the fold`*** | 展示している製品やソリューションを見るために、ユーザーにスクロールさせる |  
| ***ソリューションを検証するため、<br />Azure Certified*** など、Microsoft のブランド化ガイドラインに従ってブランド画像 を使う | 承認されていない Microsoft ブランド画像を使用している。 |  

*   Azure Certified バッジの詳細については、Microsoft Azure Certified ガイドライン ページ ([azure.microsoft.com/support/legal/marketplace/certified-guidelines](https://azure.microsoft.com/support/legal/marketplace/certified-guidelines)) を参照してください。  
*   ブランドの追加リソースについては、Brand Central のページ ([microsoft.sharepoint.com/teams/brandcentral](https://microsoft.sharepoint.com/teams/brandcentral)) を参照してください。  
*   Logo Builder ツールの詳細については、Logo Builder のページ ([logobuilder.partner.microsoft.com](https://logobuilder.partner.microsoft.com)) を参照してください。  
*   詳細については、「しくみについて」のページ ([partner.microsoft.com/membership](https://partner.microsoft.com/membership/how-it-works)) を参照してください。
