# <a name="azure-marketplace-participation-policies"></a>Azure Marketplace 参加ポリシー

これらの Microsoft Azure Marketplace 参加ポリシーは、Microsoft Azure Marketplace に参加するすべての発行元とオファリングに適用されます。 これらのポリシーは、Microsoft Marketplace Publisher Agreement に明記された使用条件に追加されるものです。 Azure Marketplace に参加するためには、発行元は常に、このドキュメントで説明されたり参照されたりしているポリシーを遵守する必要があります。 特定の時点で発行元がこれらのいずれかの使用条件を遵守しない場合、Microsoft は、その発行元のオファリングを Azure Marketplace から削除することがあります。 Microsoft は、時おり、このドキュメントを更新することがあります。

## <a name="base-criteria"></a>基本条件

1. Azure Marketplace で提供されるソフトウェアおよびサービスは、以下の基準のうち少なくとも 1 つを満たしていなければなりません。 

    * *Microsoft Azure 上で稼働する*: ソフトウェアまたはサービスの基本機能は、Microsoft Azure 上で稼働するものである必要があります。
    * *Microsoft Azure にデプロイ可能*: 発行元は、ソフトウェアまたはサービスを Microsoft Azure 上にデプロイする方法をオファリングの一覧情報の中に記述しなければなりません。
    * *Microsoft Azure サービスと統合する、または Microsoft Azure サービスを拡張する*: オファリングの一覧情報で、発行元は、次のことを説明する必要があります。ソフトウェアまたはサービスがどの Azure サービスと統合されるのか、またはどの Azure サービスを拡張するのか。 ソフトウェアまたはサービスが Azure サービスとどのように統合されるのか、または Azure サービスをどのように拡張するか。

1. 各オファリングについて、発行元は、Azure Marketplace によってサポートされる販売先の国のうち少なくとも 1 つの国でオファリングを利用可能にする必要があります。 Azure Marketplace は、現在、以下の販売先の国をサポートしています。アルジェリア、アルゼンチン、オーストラリア、オーストリア、バーレーン、ベラルーシ、ベルギー、ブラジル、ブルガリア、カナダ、チリ、コロンビア、コスタリカ、クロアチア、キプロス、チェコ共和国、デンマーク、ドミニカ共和国、エクアドル、エジプト、エルサルバドル、エストニア、フィンランド、フランス、ドイツ、ギリシャ、グアテマラ、香港特別行政区、ハンガリー、アイスランド、インド、インドネシア、アイルランド、イスラエル、イタリア、日本、ヨルダン、カザフスタン、ケニア、クウェート、ラトビア、リヒテンシュタイン、リトアニア、ルクセンブルク、マケドニア、マレーシア、マルタ、メキシコ、モンテネグロ、モロッコ、オランダ、ニュージーランド、ナイジェリア、ノルウェー、オマーン、パキスタン、パナマ、パラグアイ、ペルー、フィリピン、ポーランド、ポルトガル、プエルトリコ、カタール、ルーマニア、ロシア、サウジアラビア、セルビア、シンガポール、スロバキア、スロベニア、南アフリカ、韓国、スペイン、スリランカ、スウェーデン、スイス、台湾、タイ、トリニダード・トバゴ、チュニジア、トルコ、ウクライナ、アラブ首長国連邦、イギリス、米国、ウルグアイ、ベネズエラ。 

1. 発行元は、財政状態が良好でなければなりません。

1. 発行元が Azure Marketplace で提供するオファリングは、限定的に利用可能または一般的に利用可能であり、確立された顧客ベースを持っている必要があります。

1. Azure Marketplace 内で提供されるオファリングは、サポートまたは販売されていない製品やコンポーネントを使用したり、それらに依存したりしてはなりません。

1. 発行元は、詳細な技術的ドキュメントを使用できるようにする必要があります。  ドキュメントでは、Microsoft Azure でそのオファリングを使用する方法を説明する必要があります。  各オファリングは、その一覧情報で、そのようなドキュメントまたはそれへのリンクを提供する必要があります。

1. 発行元は、発行元の一般向け Web サイト上で Azure Marketplace でオファリングが利用可能になったことを発表し、[https://azuremarketplace.microsoft.com/marketplace/](https://azuremarketplace.microsoft.com/marketplace/) のオファー一覧ページへのハイパーリンクを記載する必要があります。

1. 発行元は、各オファリングを Microsoft が提供する分類法 (このドキュメントのセクション 4 に記載したカテゴリを含む) の 1 つ以上に基づいて分類する必要があります。 Microsoft は、発行元の選択した分類が不正確であると見なした場合に、任意のオファリングの分類を変更する権利を留保します。

1. 発行元のオファリングが Microsoft Azure Certified であるものの、主として Microsoft Azure 上で実行されるものでない場合、発行元は当該オファリングの無料版または BYOL 版を発行してから 90 日以内に Marketplace で有料製品版のオファリングを利用可能にしなければなりません。

## <a name="publishing-offers"></a>オファーの発行

1. 発行元は、Microsoft Marketplace Publisher Agreement を締結してから 60 日以内に、少なくとも 1 つのオファリングを Azure Marketplace に発行しなければなりません。

1. 発行元は、Marketplace Publication Guidelines で定義され、その後変更があった場合は発行ポータルで確認できる、公開のための Azure Marketplace 技術要件に準拠しなければなりません。

## <a name="offer-listings"></a>オファーの一覧情報

1. 発行元は、オファーの一覧情報ページにオファーの詳細な情報を含めなければならず、その情報は、正確かつ最新のものでなければなりせん。 そのような情報として、可能な場合、次のものを含める必要があります。
    * *オファリングの説明*
        * 最低限のオファーの説明
            * SKU 情報
            * 価値の提案
        * 推奨されるオファーの説明
            * 詳細な SKU 情報
            * 詳細な価値の提案
            * 機能: オファリングに関する事実の記述 3 ～ 5 件
            * メリット:オファリングの機能によって生み出される結果 3 ～ 5 件
    * *価格モデル*: オファリングは、このドキュメントの「**価格モデル**」セクションで説明されている、Azure Marketplace によってサポートされる[価格モデル](./marketplace-faq-publisher-guide.md)に適合している必要があります。
    * *カスタマー サポートの詳細へのリンク*: 発行元は、Azure Marketplace で提供するオファリングに対して、商取引上合理的なカスタマー サポートを提供しなければなりません。  オファリングに関連付けられたユーザー料金の一部として、または別個に購入するサポート オファリングとして含めることができます。
    * *オファリングの参照資料*: 参照資料には、発行元のオファリングを Microsoft Azure で使用する方法に関するデモ ビデオ、スクリーン ショット、ホワイトペーパー、事例研究、推薦情報、および詳細な技術資料が含まれます (ただし、これらに限定されません)。
    * *お客様への返金のポリシー*
    * *使用条件*
    * *プライバシー ポリシー*

1. 発行元は、オファーの一覧情報ページの中で、Azure Marketplace から入手可能でないソフトウェアまたはサービスに Azure のお客様をリダイレクトしたり、それらのソフトウェアまたはサービスをアップセルしてはなりません。 この制限は、発行元が Azure Marketplace の外部で販売しているサポート サービスには適用されません。

1. 発行元は、Azure Marketplace 内で、他のクラウド プラットフォーム上で利用可能なオファリングを販売促進してはなりません。

1. Microsoft は、オファーの一覧情報ページの詳細内容を品質保証の目的で編集および改訂する権利を留保します。 Microsoft が一覧情報ページの詳細内容に何らかの変更を加える場合は、Azure Marketplace でオファリングの一覧情報ページを発行する前に発行元に通知します。

|     |    |
| :------------------- | :-------------------|

| **仮想マシン拡張機能** | REST API、Azure portal、Azure PowerShell コマンドレットを含む多様なオプションを使用して新しい VM に追加できる VM エージェント。 VM 拡張機能は、既存の VM に手動でインストールしたり、Windows Server または Linux ベースの VM のいずれかに対して構成したりすることもできます。 <br> <br> 仮想マシン拡張機能 ("**VM 拡張機能**") は、Azure 仮想マシン内で、1 つのソフトウェア アプリケーションまたは一連のソフトウェア アプリケーションをインストールするためのメカニズムです。 VM 拡張機能には、ソフトウェア アプリケーションが含まれることがあります。  それが仮想マシン内でインストールされたり実行されると、外部の場所から 1 つ以上のソフトウェア アプリケーションがダウンロードされ、インストールされることがあります。 明確に区別するため、VM 拡張機能によってインストールされるソフトウェアや他のデータのことを、本契約では「オファリング コンテンツ」と呼ぶことにします。 VM 拡張機能オファリングに関連する VM 拡張ハンドラーとその顧客サポートについては、自己責任となります。 |
| **サービス** | インフォメーション ワーカー、ビジネス アナリスト、開発者、IT 担当者が、カスタム アプリケーション開発またはシステム管理において使用できるフル マネージド サービス。 Marketplace では、次の 3 種類のサービスをサポートしています。 <br> <br> **アプリケーション サービス**は、お客様が Azure でクラウド規模のアプリケーションを迅速に開発する機能を提供するものです。 <br> <br> アプリケーション サービスを購入するお客様は、Azure サブスクリプションを保有している必要があります。 発行元には、Microsoft Marketplace Publisher Agreement で詳述されているとおり、お客様のアプリケーション サービスの利用状況を計測し、利用状況の情報を Microsoft に報告する責任があります。|

| **カタログ リスト** | Marketplace 経由では Azure のお客様に直接提供されないが、Marketplace にはリンク、アイコン、およびソフトウェア/サービス製品の一覧が表示されるオファリング。  お客様は、発行元の Web サイトにリダイレクトされるか、Azure でのオファリングを取得して使用する方法の手順が提供されます。 "**カタログのみリスティング**" は、お客様が Marketplace 経由では直接提供されないオファリングです。 その代わり、お客様を発行元 Web サイトにリダイレクトするリンク、アイコン、説明が Marketplace に表示されます。 さらに、お客様が Azure でオファリングを入手して使用する方法に関する手順が提供されます。 明確に区別するため、"カタログのみリスティング" が指すソフトウェアやデータはすべて、本契約では「オファリング コンテンツ」と見なされます。 |

| | |

発行元は、発行済みの仮想マシン オファリングのユーザー料金を引き下げられるようになりました。 現在 Marketplace では、既存のオファリングに対するその他の種類の価格変更はすべてサポートされていません。 発行元がオファリングに関連付けられているユーザー料金を変更する場合は、最初に Marketplace からオファリングを削除する必要があります。  削除は、Microsoft Marketplace Publisher Agreement とこのドキュメントの要件に従って行う必要があります。  発行元はその後、新しいユーザー料金を含む新しいオファリングを発行できます。

発行元は、Marketplace でサービス オファリングを発行した後、独自のデータ ログを最低でも前の 2 暦年分保持する必要があります。 データ ログには、オファリングの顧客へのプロビジョニング情報を含めます。 発行元のデータ ログと Microsoft のデータ ログのデータに不一致がある場合、Microsoft のデータ ログが優先されます。

## <a name="offering-suspension-and-removal"></a>オファリングの一時中断と削除

1. Microsoft は Azure Marketplace で、任意の理由でオファリングを一時中断または削除する権利を留保します。 Microsoft がオファリングを削除する理由には、次のものが含まれます (ただし、これらに限定されません)。
    * オファリングが 6 か月以上どのお客様にもプロビジョニングされなかった。
    * オファリングの有料 SKU のキャンセル率が高い。
    * オファリングに対してお客様から一貫して否定的なフィードバックを受け取っている。
    * オファリングに対して一貫して多数のサポート チケットを受け取っている。
    * 発行元が、Microsoft Marketplace Publisher Agreement、Marketplace Publication Guidelines、またはこのドキュメントに明記された使用条件に適合しなくなった。

1. さまざまな理由から、Marketplace からのプランの削除を決定する場合があります。 プランの削除により、新しい顧客のプラン購入やデプロイがなくなる場合はありますが、既存の顧客への影響はありません。 プランの終了とは、既存の顧客との間で、サービスやライセンス契約を終了するプロセスのことです。 オファリングの削除と終了に関するガイダンスおよびポリシーは、Microsoft Marketplace Publisher Agreement で定められています (第 7 条参照)。 削除または終了は、サポート チケットを記録することで依頼できます。

## <a name="taxes"></a>税

1. **Microsoft 送金国。**

    1. 以下の国が、Microsoft Marketplace Publisher Agreement で定義されている Microsoft 送金国です。欧州連合、カナダ、インド、アイルランド、リヒテンシュタイン、モナコ、ニュージーランド、ノルウェー、プエルトリコ、韓国、スイス、米国。 オーストラリア、サウジアラビア、台湾、およびアラブ首長国連邦は、Web Direct 販売に関してのみ Microsoft 送金国となっています。

    1. Microsoft は、アイルランド国内のすべてのお客様に対して、VAT を請求します (アイルランドのお客様から有効な VAT 56B 証明書の詳細が提供された場合を除く)。 欧州連合およびノルウェー (アイルランド国外に位置する) の法人のお客様については、有効な VAT 番号が提供された場合、VAT のゼロ税率が適用されます。 有効な VAT 番号が提供されない場合、お客様の国または管轄区における適切な VAT 率が請求されます。

    1. Microsoft は、スイスまたはリヒテンシュタインのすべてのお客様にスイスの付加価値税を請求します。

1. **ISV 送金国。**

    1. ISV 送金国には、本ポリシーのセクション 1 (3) で定義されている、Microsoft 送金国以外のすべての販売先の国が含まれます。 また、Enterprise チャネル経由でのオーストラリア、サウジアラビア、台湾、およびアラブ首長国連邦への販売は、ISV 送金販売として扱われます。 ただし台湾の発行元に関しては、そのような台湾の発行元が、適用される税金の徴収と送金を行っていることを Microsoft が確認できません。

    1. 発行元が、ISV 送金国でオファリングを利用可能にすることを選択する場合、そのような ISV 送金国における納税義務の判断については、それらの発行元のみが責任を負うものとします。  Microsoft は現在、発行元が顧客に税金の請求書を提供するために必要な取引の詳細を提供できないことを、発行元が承認する必要があります。
