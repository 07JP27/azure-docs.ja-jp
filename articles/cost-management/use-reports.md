# <a name="use-cost-management-reports"></a>Cost Management レポートの使用

この記事では、Cloudyn ポータルでさまざまな Cost Management レポートを使用する方法について説明します。 ほとんどの Cloudyn レポートは、直感的で、統一感のある外観をしています。 Cloudyn レポートの概要については、「[Understanding cost reports (コスト レポートについて)](understading-cost-reports.md)」をご覧ください。 また、ほとんどのレポートで使用されるさまざまなオプションとフィールドについても、この記事で説明します。

## <a name="cost-analysis-reports"></a>コスト分析レポート

コスト分析レポートには、クラウド プロバイダーからの課金データが表示されます。 レポートを使用して、課金ファイルの明細行になっているさまざまなデータ セグメントのグループ化やドリルダウンが可能です。 レポートでは、クラウド ベンダーの生の課金データに対して詳細なコスト移動が可能です。

コスト分析レポートでは、タグ別にコストをグループ化しません。 タグに基づくレポートは、Cost Allocation 360 を使用したコスト モデルの作成後に設定される、原価配賦レポートのみで利用可能です。

### <a name="actual-cost-analysis"></a>実績コストの分析

実績コスト分析レポートは、継続的なコストや 1 回限りの料金など、主なコスト要因を表示します。

 実績コスト分析レポートは、次の目的に使用します。

- 指定された時間枠内に使用された実績コストを分析および管理する
- しきい値による警告をスケジュールする
- ショーバックおよびチャージバック コストを分析する

#### <a name="to-use-the-actual-cost-analysis-report"></a>実績コスト分析レポートを使用するには、次の手順を実行します。

少なくとも、以下の手順を実行してください。 他のオプションおよびフィールドを使用することもできます。

1. 日付範囲を選択します。
2. フィルターを選択します。

レポート結果を右クリックしてドリルダウンすると、より詳しい情報を表示できます。

![実績コスト分析レポートの例](./media/use-reports/actual-cost-analysis.png)

### <a name="actual-cost-over-time"></a>実績経過コスト

実績経過コスト レポートは、定義された期間の精度でコストを分配する標準的なコスト分析レポートです。 レポートでは、傾向を観察して支出の不均衡を検出できるように、時間の経過に沿って支出を表示します。 このレポートは、継続的なコストや 1 回限りの予約インスタンスの料金など、選択された時間枠内に使用された主なコスト要因を表示します。

実績経過コスト レポートは、次の目的に使用します。

- 時間の経過に沿ってコストの傾向を確認する。
- コストの不均衡を検出する。
- アマゾン ウェブ サービスに関わるコスト関連のあらゆる疑問点を見つける。

#### <a name="to-use-the-actual-cost-over-time-report"></a>実績経過コスト レポートを使用するには、次の手順を実行します。

少なくとも、以下の手順を実行してください。 他のオプションおよびフィールドを使用することもできます。

- 日付範囲を選択します。

たとえば、時間の経過に沿ってコストを表示するグループを選択できます。 次に、フィルターを追加して結果を絞り込みます。

![実績経過コスト レポートの例](./media/use-reports/actual-cost-over-time.png)



### <a name="amortized-cost-reports"></a>償却コスト レポート

この一連の償却コスト レポートでは、使用ベースでない線形化されたサービスの料金や、支払いが 1 回限りのコストを表示し、これらの有効期限内にコストを均等に配分します。

1 回限りの料金には、以下のような例があります。

- 年払いのサポート料金
- 年払いのセキュリティ コンポーネント料金
- 予約インスタンスの購入料金
- Azure Marketplace の一部のアイテム

課金ファイルでは、サービスの使用開始日と終了日 (タイムスタンプ) が同じ値の場合、 1 回限りの料金に対して特徴付けが行われています。 Cloudyn では、これらを償却可能な 1 回限りの料金として認識します。 オンデマンドでの使用量のコストを伴うその他の使用ベースのサービスは、償却できません。

償却コストを示すために、実績経過コスト レポートの以下の画像の例を確認しましょう。 例では、8 月 23 日にコストの急上昇が認められます。 通常の毎日のコスト傾向と比較すると、異常とも考えられます。 根本原因の分析とデータ移動では、この日に購入および課金された 1 回限りの料金である年間の AWS サービスの APN 予約として、このコストを識別しました。 次のセクションでこのコストの償却方法を確認できます。

![1 回限りのコストを示した実績経過コスト レポートの例](./media/use-reports/actual-amort-example.png)

#### <a name="to-use-the-amortized-cost-over-time-report"></a>償却経過コスト レポートを使用するには、次の手順を実行します。

少なくとも、以下の手順を実行してください。 他のオプションおよびフィールドを使用することもできます。

1. 日付範囲を選択します。
2. サービスおよびプロバイダーを選択します。

前の例からの続きとして、次の図では、1 回限りのコストが償却されていることを確認できます。

![償却経過コスト レポートの例](./media/use-reports/amort-cost-over-time.png)

上図では、時間の経過に沿って APN 予約にかかる償却コストが示されています。 このレポートでは、1 回限りの料金の償却と年間予約購入としての APN コストを示しています。 APN コストは、予約の事前コストの 365 分の 1 として、日次ベースで均等に分配されます。

## <a name="cost-allocation-analysis-reports"></a>原価配賦分析レポート

原価配賦分析レポートは、Cost Allocation 360 を使用してコスト モデルを作成した後に利用できます。 Cloudyn では、原価/課金データを処理し、クラウド アカウントの使用状況とタグ データに対してデータをマッチングします。 データをマッチングするために、Cloudyn では使用状況データへのアクセスが必要です。 資格情報が不足しているアカウントは、未分類のリソースとしてラベル付けされます。

### <a name="cost-analysis-report"></a>コスト分析レポート

コスト分析レポートでは、選択した時間枠内のクラウドの使用や支出に関する洞察を提供します。 Cost Allocation Manager のポリシー セットは、コスト分析レポートで使用されます。

Cloudyn によるこのレポートの計算方法

Cloudyn では、会計アフィニティを適用して、リンク付けされている各会計の整合性を保持します。 アフィニティでは、特定のサービスを使用しない会計は、その会計に配賦されているこのサービスのコストを保持しないことを保証します。 該当の会計で未払いになっているコストは、その会計に残されたままになり、配賦ポリシーでは計算されません。 たとえば、リンク付けされている会計が 5 つあるとします。 そのうちの 3 つだけがストレージ サービスを使用している場合、ストレージ サービスのコストは 3 つの会計のタグにのみ配賦されます。

 コスト分析レポートは、次の目的で使用します。

- 特定の時間枠内のデプロイ全体の集計ビューを表示する。
- コスト モデルで作成されたポリシーに基づいて、タグ分類ごとにコストを表示する。

#### <a name="to-use-the-cost-analysis-report"></a>コスト分析レポートを使用するには、次の手順を実行します。

1. 日付範囲を選択します。
2. 必要に応じて、タグを追加します。
3. グループを追加します。
4. 事前に作成したコスト モデルを選択します。

次の図は、サンバースト形式のコスト分析レポートの例を示しています。 リングはグループを示しています。 外側のリングはサービスを示し、内側の円は単位を示しています。

### <a name="cost-over-time-report"></a>時間経過コスト レポート

時間経過コスト レポートでは、傾向を観察してデプロイの不均衡を検出できるように、時間の経過に沿って支出を表示します。 本質的に、定義された期間に分配されたコストを表示します。 レポートには、継続的なコストや 1 回限りの予約インスタンスの料金など、選択された時間枠内に使用された主なコスト要因が含まれます。 このレポートでは、Cost Manager 360° に設定されたポリシーを使用できます。

時間経過コスト レポートは、次の目的に使用します。

- 時間の経過に沿った変化と、該当の日 (日付範囲) から次の日への変化に影響を与えた要因を確認する。
- 特定のインスタンスに対して、時間の経過に沿ってコストを分析する。
- 特定のインスタンスでコストの増加が発生した理由を理解する。

#### <a name="to-use-the-cost-over-time-report"></a>時間経過コスト レポートを使用するには、次の手順を実行します。

1. 日付範囲を選択します。
2. 必要に応じて、タグを追加します。
3. グループを追加します。
4. 事前に作成したコスト モデルを選択します。
5. 実績コストまたは償却コストを選択します。
6. 生の課金データ ビューを表示するために配賦ルールを割り当てるか、または Cloudyn ビューによる再計算されたコストに対して割り当てるかを選択します。
