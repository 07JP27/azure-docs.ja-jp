# <a name="azure-active-directory-b2c-sign-up-and-sign-in-consumers-in-your-applications-using-custom-policies"></a>Azure Active Directory B2C: サインアップして、カスタム ポリシーを使用して、アプリケーションでコンシューマーにサインイン
カスタム ポリシーは、Azure AD B2C テナントの動作を定義する構成ファイルです。 それらは、ほぼ無制限の数のタスクを完了する、identity 開発者によって完全に編集することがことができます。

## <a name="how-to-articles"></a>操作方法に関する記事
Azure Active Directory B2C の特定の機能を使用する方法を学習します。

* [作業の開始](active-directory-b2c-overview-custom.md)
* OIDC プロバイダーを構成するように[Azure AD](active-directory-b2c-setup-aad-custom.md)です。
* SAML プロバイダーを構成するように[Salesforce](active-directory-b2c-setup-sf-app-custom.md)です。
* RESTful Api を統合します。
    * [別のクレームを取得](active-directory-b2c-rest-api-step-custom.md)です。
    * [ユーザー入力を検証](active-directory-b2c-rest-api-validation-custom.md)です。
* ログインをカスタマイズします。
    * [ユーザー入力を構成します。](active-directory-b2c-configure-signup-self-asserted-custom.md)
    * [UI をカスタマイズします。](active-directory-b2c-ui-customization-custom.md)
    * [トークンをカスタマイズします。](active-directory-b2c-reference-manage-sso-and-token-configuration.md)
* トラブルシューティングを行う[Application Insights を使用してログを収集](active-directory-b2c-troubleshoot-custom.md)です。
